# Pokémon Generation 9 (Scarlet/Violet) Battle Mechanics Specification

## Overview

Generation 9 (Scarlet/Violet) represents the current modern standard for Pokémon battle mechanics. Key characteristics:

- **Terastallization** replaces Dynamax as the primary battle gimmick
- **Snow** weather replaces Hail with new defensive benefits for Ice-types
- **Open-world mechanics** including Auto-Battle (Let's Go) feature
- **Complete regional Pokédex** with all Pokémon available via DLC (The Hidden Treasure of Area Zero)
- No Mega Evolution, Z-Moves, or Dynamax—only Terastallization remains
- Maintains the 18-type system from Gen 6+
- Critical hit rate: 1/24 base (same as Gen 6+)

This is the authoritative ruleset for modern competitive Pokémon battles. All prior rulesets (Gen 3–8) are superseded by Gen 9 mechanics when in effect.

## Terastallization

### Concept

Terastallization is a transformative battle gimmick where a Pokémon's type temporarily changes to its **Tera Type**. Unlike Dynamax:

- **Permanent for the battle**: Once a Pokémon Terastallizes, it remains Tera-typed for the entire battle (does not revert)
- **One per battle per team**: Each trainer can use Terastallization once per battle on any of their Pokémon
- **Single type, not dual**: The Pokémon becomes a single Tera Type, replacing its original type(s)
- **Visual effect**: The Pokémon sparkles with crystalline jewels matching the Tera Type
- **Strategic flexibility**: Common use cases include pure Fire-types gaining Water typing for coverage, or tanky Pokémon shifting to neutral matchups

### Tera Type Mechanics

**Setting Tera Type**: Each Pokémon has a Tera Type assigned before battle. In competitive, this is typically revealed to the opponent at team preview.

**Type Change on Terastallization**:
```typescript
interface Pokémon {
  originalTypes: Type[]; // e.g., ["Fire", "Flying"] for Charizard
  teraType: Type;        // e.g., "Water"
  terastallized: boolean;
  activateTerastallization(): void {
    this.terastallized = true;
    // currentType() now returns teraType
  }
}

currentType(): Type {
  return this.terastallized ? this.teraType : this.originalTypes[0];
  // For type interactions, use teraType if terastallized
}
```

### STAB (Same-Type Attack Bonus) Interaction with Tera

The STAB calculation becomes complex when Terastallization is active. The rule is:

1. **Tera Type matches one of the original types** (e.g., Fire/Flying Charizard with Fire Tera):
   - That type gets **2x STAB** (increased from normal 1.5x)
   - Other original type (if applicable) loses STAB

2. **Tera Type is different from both original types** (e.g., Fire/Flying Charizard with Grass Tera):
   - Tera Type gets **1.5x STAB** (normal STAB multiplier)
   - Original types keep **1.5x STAB** (unchanged)
   - This allows stacking two different STABs simultaneously

3. **Single-type Pokémon with different Tera Type** (e.g., Fire Charizard with Dragon Tera):
   - Tera Type gets **1.5x STAB**
   - Fire type loses STAB

### STAB Calculation Examples

**Example 1: Charizard (Fire/Flying) with Fire Tera using Flamethrower**
```
Flamethrower is Fire-type.
Charizard is terastallized to Fire.
Fire matches one original type.
Result: 2x STAB applied
Damage = Base × 2.0 (STAB)
```

**Example 2: Charizard (Fire/Flying) with Grass Tera using Energy Ball**
```
Energy Ball is Grass-type.
Charizard is terastallized to Grass.
Grass does NOT match either original type.
Result: Tera Type gets 1.5x STAB, original types inert
Damage = Base × 1.5 (STAB)
```

**Example 3: Charizard (Fire/Flying) with Grass Tera using Flamethrower**
```
Flamethrower is Fire-type.
Charizard is terastallized to Grass.
Fire matches an original type, but Charizard is now Grass-type.
Result: Original types keep STAB (1.5x) even though Pokémon is Grass
Damage = Base × 1.5 (original STAB)
```

**Example 4: Charizard (Fire/Flying) with Grass Tera, ability Adaptability**
```
Adaptability doubles STAB from 1.5x to 3.0x.
For Flamethrower (Fire):
  Base STAB = 1.5x (original type kept)
  With Adaptability = 3.0x
For Energy Ball (Grass):
  Base STAB = 1.5x (Tera type)
  With Adaptability = 3.0x
```

### Tera Blast Move

**Tera Blast** is the signature move enabled by Terastallization:

```typescript
interface TeraBlastMove extends Move {
  basePower: 80;
  type: () => Type {
    // On a terastallized Pokémon, becomes the Tera Type
    if (user.terastallized) {
      return user.teraType;
    }
    // On a non-terastallized Pokémon (rare), uses Normal
    return "Normal";
  };
  attackStat: () => Stat {
    // Uses the higher of Attack or Special Attack
    return user.attack > user.spAtk ? "Attack" : "SpAtk";
  };
  accuracy: 100;
  priority: 0;
  makeContact: false;
}
```

**Tera Blast Behavior**:
- Power: 80
- Accuracy: 100%
- Category: Physical or Special based on user's higher offensive stat
- Type: Always matches the user's current type (if terastallized, their Tera Type)
- No contact made
- Effect: None beyond damage

**STAB Application with Tera Blast**:
When a terastallized Pokémon uses Tera Blast:
- The move becomes the Tera Type
- STAB rules apply using the Tera Type as reference
- Always gets at least 1.5x STAB (since it's always the Pokémon's current type)

## Snow Replaces Hail

### Weather Change

In Generation 9, **"Hail"** weather is renamed **"Snow"** with new mechanics:

```typescript
enum WeatherType {
  NONE = "None",
  RAIN = "Rain",
  SAND = "Sand",      // Sandstorm
  SUN = "Sun",         // Sunny weather
  SNOW = "Snow",       // Replaces Hail (Gen 6-8)
  HARSH_SUNLIGHT = "HarshSunlight",
  HEAVY_RAIN = "HeavyRain",
}

interface Weather {
  type: WeatherType;
  duration: number; // turns remaining, or -1 for indefinite (Desolate Land, etc.)
}
```

### Snow Effect Changes

**Old Hail (Gen 3-8)**:
- All Pokémon except Ice-types take 1/8 damage per turn
- Ice-types are immune to this damage

**New Snow (Gen 9)**:
- **Removes residual damage** to non-Ice types
- **Boosts Ice-type Defense by 50%** (multiplicative)
  ```typescript
  // In defense calculation:
  if (isWeather("Snow") && defender.type.includes("Ice")) {
    defenseMultiplier *= 1.5;
  }
  ```
- Ice-types gain a concrete defensive benefit instead of just immunity

### Abilities That Reference Snow

Abilities that previously referenced "Hail" now reference "Snow":

| Ability | Effect |
|---------|--------|
| **Hail** (move effect only) | Sets up Snow weather for 5 turns |
| **Ice Body** | Heals 1/8 HP per turn in Snow |
| **Snow Warning** | Sets up Snow weather upon entry |
| **Snow Cloak** | Raises evasion in Snow (still present) |
| **Slush Rush** | Doubles Speed in Snow (unchanged mechanics) |

## Damage Formula

### Complete Damage Formula (Gen 9)

Generation 9 uses the damage formula foundation from Gen 3+ with a Tera modifier addition:

```
Damage = ((((2 × Level / 5 + 2) × Power × Attack / Defense) / 50) + 2) × Modifiers
```

### Modifier Chain (Exact Order)

Applied in this specific sequence:

1. **STAB** (1.5x or 2.0x based on Tera rules above)
2. **Type Effectiveness** (0.25, 0.5, 1.0, 2.0, or 4.0)
3. **Weather** (Sun boosts Fire 1.5x, Rain boosts Water 1.5x, etc.)
4. **Abilities** (Flash Fire, Dry Skin, Filter, Tinted Lens, etc.)
5. **Items** (Life Orb +1.3x, Choice Band +1.5x, etc.)
6. **Generation-Specific Modifiers**:
   - Critical hit: 1.5x (after all other modifiers in Gen 6+)
7. **Random Factor**: 0.85 to 1.0 (15% damage variance)
8. **Screens** (Reflect -1.0x, Light Screen -1.0x or -0.75x depending on physical/special)
9. **Burn** (Physical moves: 0.5x if attacker is burned, does not affect Special Attacks)
10. **Final Damage**

### Tera Modifier Placement

The Tera Type change affects:
- **STAB calculation** (see Terastallization section)
- **Type Effectiveness** (the move's type doesn't change, but matchup changes):
  ```typescript
  // Example: Terastallized Grass Pokémon takes Water move
  // Pokémon's type changed to Grass, so Water is now super-effective
  typeEffectiveness = getTypeMatchup(moveType, pokémonCurrentType);
  ```
- **Ability interactions** (some abilities key off type)

### Damage Calculation Code Example

```typescript
function calculateDamage(
  attacker: Pokémon,
  defender: Pokémon,
  move: Move,
  battle: Battle
): number {
  const level = attacker.level;
  const power = move.basePower;
  const attackStat = move.category === "Physical" ? attacker.attack : attacker.spAtk;
  const defenseStat = move.category === "Physical" ? defender.defense : defender.spDef;

  // Base damage
  const baseDamage =
    Math.floor((((2 * level) / 5 + 2) * power * attackStat) / defenseStat / 50) + 2;

  // Modifiers
  let multiplier = 1.0;

  // 1. STAB
  multiplier *= calculateStab(attacker, move);

  // 2. Type Effectiveness
  multiplier *= getTypeMatchup(move.type, defender.currentType());

  // 3. Weather
  if (battle.weather.type === "Sun" && move.type === "Fire") {
    multiplier *= 1.5;
  } else if (battle.weather.type === "Rain" && move.type === "Water") {
    multiplier *= 1.5;
  }
  // ... other weather effects

  // 4. Abilities (attacker/defender)
  multiplier *= applyAbilityModifiers(attacker, defender, move);

  // 5. Items
  multiplier *= applyItemModifiers(attacker, move);

  // 6. Critical hit
  const isCritical = rollCritical(attacker, move);
  if (isCritical) {
    multiplier *= 1.5;
  }

  // 7. Random variance
  const randomFactor = Math.random() * 0.15 + 0.85; // 0.85-1.0
  multiplier *= randomFactor;

  // 8. Screens
  multiplier *= applyScreens(defender, move.category, battle);

  // 9. Burn (physical only)
  if (
    attacker.hasStatus("Burn") &&
    move.category === "Physical" &&
    !attacker.ability.includes("Guts")
  ) {
    multiplier *= 0.5;
  }

  return Math.max(1, Math.floor(baseDamage * multiplier));
}
```

## No Dynamax, Mega Evolution, or Z-Moves

### Removed Mechanics

Effective in Generation 9:

- **Dynamax** is completely removed (was active in Gen 8)
- **Mega Evolution** is removed (was active in Gen 6-7)
- **Z-Moves** are removed (was active in Gen 7)

### Why Only Terastallization?

Design philosophy: Terastallization provides strategic flexibility without power creep. It allows type-flexibility plays (offensively and defensively) without permanently buffing stats like Dynamax or Mega Evolution did.

### Implementation

## Critical Hit System

### Unchanged from Gen 6+

Gen 9 maintains the critical hit mechanics from Generation 6 onward:

```typescript
interface CriticalHitModifier {
  baseRate: 1 / 24,      // ~4.17% base chance
  stage0: 1 / 24,        // Default
  stage1: 1 / 8,         // High crit ratio moves (e.g., Scope Lens, Focus Energy)
  stage2: 1 / 2,         // Stage 2 crit (rare)
  stage3: 1 / 1,         // Guaranteed crit (move effect or ability)
}

function isCritical(attacker: Pokémon, move: Move): boolean {
  let stage = 0;

  // Attacker's crit stage
  stage += attacker.critStage;

  // Move's crit stage
  stage += move.critStage || 0; // e.g., Scope Lens adds 1

  // Ability modifiers
  if (attacker.ability === "Super Luck") {
    stage += 1;
  }

  // Item modifiers
  if (attacker.heldItem === "Scope Lens") {
    stage += 1;
  }

  // Get rate based on stage
  const rate = criticalHitTable[Math.min(stage, 3)];
  return Math.random() < rate;
}

function applyDamageModifierForCrit(
  baseDamage: number,
  isCrit: boolean
): number {
  // Critical hit always multiplies by 1.5x in Gen 6+ (including Gen 9)
  return isCrit ? baseDamage * 1.5 : baseDamage;
}
```

### Critical Hit Calculation Order

Critical hits are applied **after** all other modifiers in the damage formula (as of Gen 6+):

```
Final Damage = Base Damage × All Modifiers (STAB, Type, Weather, etc.) × (isCritical ? 1.5 : 1.0)
```

## End-of-Turn Order

### Execution Sequence

At the end of each turn, the following occur in **strict order**:

1. **Ability end-of-turn triggers**
   - Abilities that activate at turn end (e.g., **Levitate** checks, **Atmospheric Override**)

2. **Weather damage and effects**
   - Sun/Rain/Sand/Snow effects
   - Snow now: boosts Ice defense (static), does NOT deal damage to non-Ice types
   - Sandstorm: deals 1/8 damage to non-Rock/Ground/Steel
   - Hail (if present): deals 1/8 damage to non-Ice (Gen 8 only; Gen 9 has Snow instead)

3. **Terrain effects**
   - Electric Terrain: halves speed of Pokémon not benefiting, regains 1/8 HP of Electric types
   - Misty Terrain: reduces damage to grounded Pokémon
   - Grassy Terrain: heals 1/8 HP to grounded Pokémon
   - Psychic Terrain: extends by 1 turn (if active), priority move immunity

4. **Entry hazard damage** (if applicable for last-switched Pokémon)
   - Spikes: 1/8, 1/6, or 1/4 depending on layers
   - Stealth Rock: 1/4 to types weak to Rock
   - Sticky Web: no damage, but Pokémon speed reduced

5. **Volatile status damage**
   - Burn: no damage (Gen 5+), but Attack reduced
   - Poison: 1/8 damage per turn
   - Badly Poisoned: escalating damage (Gen 9 uses standard poison)
   - Salt Cure: 1/8 or 1/4 damage (see Status Changes section)
   - Leech Seed: drains 1/8 HP to seeder

6. **Screen/Reflect countdown**
   - Reflect reduces by 1
   - Light Screen reduces by 1
   - Both disappear when duration hits 0

7. **Weather duration reduction**
   - Permanent weather (Desolate Land, Primordial Sea, Delta Stream) do not reduce
   - Temporary weather (Rain, Sun, Sandstorm, Snow) reduce by 1 turn

8. **Terrain duration reduction**
   - Electric Terrain, Misty Terrain, Grassy Terrain, Psychic Terrain all reduce by 1 turn

### Snow-Specific End-of-Turn

```typescript

    if (pokemon.hasType("Ice")) {

```

**Key Difference from Hail**: Snow does NOT deal damage. The 1/8 residual damage to non-Ice types is completely removed.

## STAB Interaction with Tera (Detailed Scenarios)

### Canonical STAB Rules with Tera

The STAB multiplier depends on whether the move's type matches the Pokémon's **current type** (Tera Type if terastallized):

```typescript
function calculateStab(
  attacker: Pokémon,
  move: Move
): number {
  const currentType = attacker.terastallized ? attacker.teraType : attacker.originalTypes[0];
  const moveType = move.type;

  // Check if move matches current type
  const isStabByCurrentType = moveType === currentType;

  // Check if move matches original types
  const isStabByOriginalType = attacker.originalTypes.includes(moveType);

  // Rule: Only boost if move matches current type or original types
  if (isStabByCurrentType) {
    // Move matches current type
    if (attacker.terastallized) {
      // If Tera Type matches an original type, boost to 2x
      if (attacker.originalTypes.includes(currentType)) {
        return 2.0;
      }
      // If Tera Type is new, use normal 1.5x
      return 1.5;
    } else {
      // No terastallization, normal STAB
      return 1.5;
    }
  }

  // If move matches original type but not current type
  if (isStabByOriginalType && attacker.terastallized) {
    // Terastallized to a different type, but move is original type
    return 1.5; // Original STAB still applies
  }

  // No STAB
  return 1.0;
}
```

### Ability Interaction: Adaptability

**Adaptability** doubles the STAB multiplier:

```typescript
function calculateStab(attacker: Pokémon, move: Move): number {
  let stabMult = calculateBaseStab(attacker, move);

  if (attacker.ability === "Adaptability") {
    // Double the STAB multiplier
    if (stabMult === 1.5) {
      stabMult = 3.0;
    } else if (stabMult === 2.0) {
      stabMult = 4.0;
    }
  }

  return stabMult;
}
```

### Gen9Ruleset Class

```typescript
import { BaseRuleset } from "./base-ruleset";
import { BattleGimmick, Move, Pokémon, Type } from "./types";

export class Gen9Ruleset extends BaseRuleset {
  name = "Generation 9 (Scarlet/Violet)";
  generation = 9;

  // Allowed battle gimmicks
  allowedGimmicks = [BattleGimmick.TERASTALLIZATION];

  // Disable older gimmicks
  canUseDynamax(): boolean {
    return false;
  }

  canMegaEvolve(): boolean {
    return false;
  }

  canUseZMove(): boolean {
    return false;
  }

  // Override: Snow replaces Hail
  getWeatherEffects(weatherType: string): WeatherEffect[] {
    if (weatherType === "Snow") {
      return [
        {
          apply: (pokemon: Pokémon) => {
            if (pokemon.hasType("Ice")) {
              // Ice-type Defense boost
              pokemon.defenseMultiplier *= 1.5;
            }
            // No residual damage to non-Ice types
          }
        }
      ];
    }
    return super.getWeatherEffects(weatherType);
  }

  // Override: STAB with Tera
  calculateStab(attacker: Pokémon, move: Move): number {
    const currentType = attacker.terastallized
      ? attacker.teraType
      : attacker.originalTypes[0];
    const moveType = move.type;

    let stab = 1.0;

    if (moveType === currentType) {
      if (attacker.terastallized && attacker.originalTypes.includes(currentType)) {
        // Tera matches original type
        stab = 2.0;
      } else {
        // Normal STAB or current type with new Tera
        stab = 1.5;
      }
    } else if (attacker.terastallized && attacker.originalTypes.includes(moveType)) {
      // Move matches original type even though Tera-typed
      stab = 1.5;
    }

    // Adaptability
    if (attacker.ability === "Adaptability" && stab > 1.0) {
      stab *= 2;
    }

    return stab;
  }

  // Override: Terastallization gimmick
  canTerastallize(pokemon: Pokémon): boolean {
    return !pokemon.terastallized && !this.terastallizationUsed;
  }

  activateTerastallization(pokemon: Pokémon): void {
    pokemon.terastallized = true;
    this.terastallizationUsed = true;
    // Broadcast visual effect, etc.
  }

  // Damage formula remains the same as Gen 3+ with Tera modifier handling
  calculateDamage(
    attacker: Pokémon,
    defender: Pokémon,
    move: Move,
    battle: Battle
  ): number {
    // Use base Gen 3+ formula with STAB override and Tera type checking
    return super.calculateDamage(attacker, defender, move, battle);
  }
}
```
