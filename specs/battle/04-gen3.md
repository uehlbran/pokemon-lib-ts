## 1. Stat System

### Physical/Special Split (Type-Based)

In Gen 3 (and Gen 1-3), whether a move is physical or special is determined by its **type**, not by the individual move. This is critical for damage calculation.

**Physical types**: normal, fighting, poison, ground, flying, bug, rock, ghost, steel
**Special types**: fire, water, electric, grass, ice, psychic, dragon, dark

All moves of a physical type use Attack/Defense. All moves of a special type use Sp.Atk/Sp.Def. There are no exceptions in Gen 3.

### Stat Formula (Gen 3+ — "Modern" Formula)

This formula is used by `BaseRuleset` and inherited by all later generations unless overridden.

#### HP Stat Formula
```typescript
HP = floor(((2 * Base + IV + floor(EV / 4)) * Level) / 100) + Level + 10
```

**Shedinja exception**: HP is always 1 regardless of base stat, IVs, EVs, or level.

#### Non-HP Stat Formula (Atk, Def, SpA, SpD, Spe)
```typescript
Stat = floor((floor(((2 * Base + IV + floor(EV / 4)) * Level) / 100) + 5) * NatureMod)
```

**Parameters**:
- `Base` = species base stat (e.g., Charizard HP base = 78)
- `IV` = Individual Value, range 0–31
- `EV` = Effort Value, range 0–252 per stat, 510 total across all stats
- `Level` = Pokémon level, range 1–100
- `NatureMod` = 1.1 (boosting), 0.9 (reducing), or 1.0 (neutral)

**Truncation**: `floor()` at each marked position. The inner calculation floors before adding +5 or +Level+10. The outer floors after nature multiplication.

#### Verification Examples

**Charizard, Lv50, 31 IVs, 0 EVs, Neutral nature** (base: HP 78, Atk 84, Def 78, SpA 109, SpD 85, Spe 100):
```
HP  = floor(((2*78 + 31 + 0) * 50) / 100) + 50 + 10 = floor(9350/100) + 60 = 93 + 60 = 153
Atk = floor((floor(((2*84 + 31 + 0) * 50) / 100) + 5) * 1.0) = floor((floor(9950/100) + 5) * 1.0) = floor(99 + 5) = 104
Def = floor((floor(((2*78 + 31 + 0) * 50) / 100) + 5) * 1.0) = floor(93 + 5) = 98
SpA = floor((floor(((2*109 + 31 + 0) * 50) / 100) + 5) * 1.0) = floor(124 + 5) = 129
SpD = floor((floor(((2*85 + 31 + 0) * 50) / 100) + 5) * 1.0) = floor(100 + 5) = 105
Spe = floor((floor(((2*100 + 31 + 0) * 50) / 100) + 5) * 1.0) = floor(115 + 5) = 120
```

### IV System
- Range: 0–31 per stat (6 independent values: HP, Atk, Def, SpA, SpD, Spe)
- Default (if unspecified in tests): 31 across all stats
- New in Gen 3 — replaces the Gen 1-2 DV system (0-15, with HP derived from other DVs)

### EV System
- Per-stat cap: 252
- Total cap: 510 across all stats
- `floor(EV / 4)` contributes to stat calculation (so 252 EV = +63 stat points)
- EVs gained from defeating wild/trainer Pokémon (species-specific yields)
- Macho Brace: doubles EV gain
- Pokérus: doubles EV gain (stacks with Macho Brace for 4x)

**Neutral natures** (no stat modification): Hardy, Docile, Serious, Bashful, Quirky

### Key Immunities
- Normal and Fighting → Ghost: **0x** (Ghost is immune)
- Ghost → Normal: **0x** (Normal is immune)
- Ground → Flying: **0x** (Flying is immune)
- Electric → Ground: **0x** (Ground is immune)
- Psychic → Dark: **0x** (Dark is immune)
- Poison → Steel: **0x** (Steel is immune)

### Dual-Type Effectiveness
For dual-typed defenders, multiply effectiveness of each type:
- 2x × 2x = **4x** (double super effective)
- 2x × 1x = **2x**
- 2x × ½x = **1x** (neutral)
- 2x × 0x = **0x** (immunity overrides)
- ½x × ½x = **¼x** (double resist)

## 3. Damage Formula

### Base Damage Calculation

```typescript
// Step 1: Level component
levelMult = floor(2 * level / 5) + 2

// Step 2: Base damage (integer division at each step)
baseDamage = floor(floor(levelMult * power * attackStat / defenseStat) / 50) + 2
```

**Notes**:
- `attackStat` and `defenseStat` are the effective stats after stat stage modifiers
- For physical-type moves: use Attack/Defense
- For special-type moves: use Sp.Atk/Sp.Def
- Integer division truncates at each step

### Modifier Chain

After base damage is calculated, modifiers are applied in this order. Each modifier is multiplied and then floored (integer truncation at each step).

<!-- VERIFY_POKEEMERALD: Exact modifier application order. The order below is from Bulbapedia Gen III documentation and Showdown Gen 3 implementation. Verify against pokeemerald's `src/battle_script_commands.c` or equivalent. -->

```
1. Weather modifier
2. Badge boost (in-game only, not competitive — skip for our engine)
3. Critical hit (2x)
4. Random factor (85-100)/100
5. STAB (1.5x)
6. Type effectiveness (applied per defender type, floor between each)
7. Burn halving (physical moves only, if attacker is burned)
```

**Important**: Type effectiveness is applied **sequentially** for dual-typed defenders, with `floor()` between each type's multiplier. This matches Gen 1-2 behavior and differs from Gen 5+ where effectiveness is combined.

```typescript
// Type effectiveness for dual-typed defender
damage = floor(damage * typeChart[moveType][defenderType1])
damage = floor(damage * typeChart[moveType][defenderType2])
```

### Weather Modifiers
- Rain + Water move: **1.5x** damage
- Rain + Fire move: **0.5x** damage
- Sun + Fire move: **1.5x** damage
- Sun + Water move: **0.5x** damage
- Sandstorm/Hail: no direct damage modifiers to moves

### STAB (Same-Type Attack Bonus)
If the move's type matches either of the attacker's types: **1.5x** damage.

### Critical Hit Effects on Damage
- **2x** damage multiplier
- Ignores attacker's negative stat stages (Attack/Sp.Atk drops)
- Ignores defender's positive stat stages (Defense/Sp.Def boosts)
- Ignores Reflect and Light Screen
- Does NOT ignore attacker's positive stages or defender's negative stages

### Burn
If the attacker is burned and the move is physical (determined by type, not individual move): damage is **halved** (0.5x).

### Random Factor
Integer from 85 to 100 inclusive (16 possible values), divided by 100. Applied as:
```typescript
damage = floor(damage * randomInt(85, 100) / 100)
```

### Spread Move Penalty (Double Battles)
In doubles, moves targeting **both foes** (Heat Wave, Rock Slide, etc.) deal 0.5x damage to each target. Moves that hit **all Pokémon** including ally (Earthquake, Explosion) do NOT have a damage penalty. See §6 for details.

### Special Cases
- **Fixed damage moves** (Seismic Toss, Night Shade): damage = attacker's level. Ignores all modifiers.
- **OHKO moves** (Horn Drill, Guillotine, Fissure, Sheer Cold): instant KO if it hits. Blocked by Sturdy.
- **Dragon Rage**: always deals exactly 40 damage.
- **SonicBoom**: always deals exactly 20 damage.

### Full Damage Calculation Example

Salamence (Lv100, Atk 394, type: dragon/flying) uses Earthquake (power 100, type: ground, physical) vs Gyarados (Def 192, type: water/flying), no weather, no items, no crit:

```
levelMult = floor(2 * 100 / 5) + 2 = 42
baseDamage = floor(floor(42 * 100 * 394 / 192) / 50) + 2
           = floor(floor(8610000 / 192) / 50) + 2
           = floor(floor(44843.75) / 50) + 2
           = floor(44843 / 50) + 2
           = floor(896.86) + 2
           = 896 + 2 = 898

No weather: × 1.0
No crit: × 1.0
Random factor: × (85-100)/100
No STAB (Salamence is dragon/flying, Earthquake is ground): × 1.0
Type effectiveness: ground → water = 1x, ground → flying = 0x
damage = floor(898 * 0) = 0
```

(Earthquake can't hit Gyarados because Flying is immune to Ground.)

## 6. Double Battles

### Overview
Double Battles (2v2) were introduced in Gen 3. Each side has two active Pokémon.

### Targeting Categories
- **Single-target**: Targets one adjacent Pokémon (most damaging moves)
- **Both opponents**: Targets both opposing Pokémon (e.g., Blizzard, Heat Wave, Rock Slide, Surf)
- **All adjacent**: Targets all adjacent Pokémon including ally (e.g., Earthquake, Explosion, Magnitude)
- **Self**: Affects the user only (e.g., Protect, Swords Dance)
- **Ally**: Targets partner (e.g., Helping Hand)
- **User's side**: Affects user's side of field (e.g., Reflect, Light Screen)

### Spread Move Damage
**Not a blanket 0.5x** — the penalty depends on targeting category:
- Moves targeting **both foes** only (e.g., Heat Wave, Rock Slide, Blizzard): **0.5x** damage to each target
- Moves targeting **all Pokémon on field** including ally (e.g., Earthquake, Explosion, Surf): **no damage reduction** (100% to each target)

Gen 4 changed the "both foes" penalty to 0.75x.
<!-- VERIFY_POKEEMERALD: Confirm this distinction. Smogon/Bulbapedia documentation supports it, but verify against pokeemerald's targeting tables. -->

### Turn Order
1. Determine each Pokémon's action (move, switch, item)
2. Switching always happens before moves
3. Among moves: sort by priority bracket (higher priority goes first)
4. Within same priority: sort by Speed (higher Speed goes first)
5. Speed ties: random 50/50

### Mid-Turn Replacement

## 7. Weather System

### Rain
- **Source**: Drizzle (permanent) or Rain Dance (5 turns)
- **Move effects**: Water moves 1.5x, Fire moves 0.5x
- **Thunder**: 100% accuracy in rain (bypasses accuracy check)
- **SolarBeam**: Takes 2 turns AND does 50% damage (half power)

- **Source**: Drought (permanent) or Sunny Day (5 turns)

- **Freeze**: Cannot be frozen while Sun is active

- **Ability effects**: Chlorophyll (2x Speed)
- **Recovery moves**: Morning Sun, Synthesis, Moonlight heal 2/3 max HP (increased from normal 1/2)

### Sandstorm
- **Source**: Sand Stream (permanent) or Sandstorm move (5 turns)
- **Damage**: 1/16 max HP per turn to all Pokémon that are NOT Rock, Ground, or Steel type
- **No move power boost**: Sandstorm does NOT boost any move type's power
- **No SpDef boost**: Rock-type SpDef boost was introduced in Gen 4, NOT Gen 3
<!-- VERIFY_POKEEMERALD: Confirm no Rock SpDef boost in Gen 3. This is critical — many sources incorrectly attribute this to Gen 3. -->
- **Ability effects**: Sand Veil (+20% evasion)
- **Recovery moves**: Morning Sun, Synthesis, Moonlight heal only 1/4 max HP
- **Weather Ball**: Becomes Rock-type with 100 base power

### Hail
- **Source**: Hail move (5 turns). **No ability** summons Hail in Gen 3 (Snow Warning is Gen 4+)
- **Damage**: 1/16 max HP per turn to all Pokémon that are NOT Ice type
- **Blizzard accuracy**: NOT boosted in Gen 3 (100% accuracy in Hail is a Gen 4 mechanic)
- **No move power boost**: Hail does NOT boost Ice move power
- **Recovery moves**: Morning Sun, Synthesis, Moonlight heal only 1/4 max HP

- Magic Guard (NOT in Gen 3 — Gen 4)

## 8. Status Conditions

### Non-Volatile Status (One at a time, persists after battle)

#### Burn
- **Damage**: 1/8 max HP per turn (at end of turn)
- **Attack reduction**: Physical move damage halved (0.5x multiplier on damage, NOT a stat drop)
- **Application**: Fire-type Pokémon cannot be burned (immune)
- **Guts interaction**: Guts nullifies the damage penalty (Attack boost applies, burn damage still occurs)

#### Paralysis
- **Speed reduction**: Speed reduced to **25% of normal** (multiplied by 0.25)
<!-- VERIFY_POKEEMERALD: Some sources say 75% reduction (speed × 0.25), others say 50% reduction. Verify against pokeemerald. Gen 7+ changed this to 50%. -->
- **Full paralysis**: 25% chance per turn to be unable to act
- **Application**: Electric-type Pokémon are immune to paralysis from **Electric-type moves only** in Gen 3. They CAN be paralyzed by non-Electric sources (Body Slam, Stun Spore, Tri Attack, etc.). Full blanket Electric-type paralysis immunity was introduced in Gen 6.

#### Poison (Regular)
- **Damage**: 1/8 max HP per turn (at end of turn)
- **Application**: Poison and Steel-type Pokémon cannot be poisoned (immune)

#### Poison (Badly Poisoned / Toxic)
- **Damage**: Starts at 1/16 max HP, increases by 1/16 each turn (1/16, 2/16, 3/16, ...)
- **Toxic counter**: Resets to 0 when the Pokémon switches out
- **Application**: Same immunities as regular poison
- **Cap**: No cap on escalation, but the counter resets on switch

#### Sleep
- **Duration**: 2–5 turns in Gen 3 (Pokémon is unable to move for 1–4 turns; the counter decrements before the action check, so a sleep count of 1 = wake on that turn and can act)
<!-- Verified: 2-5 turns per Bulbapedia Gen 3 sleep mechanics -->
- **Wake turn**: Pokémon CAN act on the turn it wakes up (unlike Gen 1)
- **Sleep Talk/Snore**: Can be used while asleep
- **Early Bird ability**: Halves sleep duration (rounds down)

#### Freeze
- **Thaw chance**: 20% per turn (1/5) — confirmed for Gen 3 (Gen 2 was 25/256 ≈ 9.77%, different rate)
- **Fire-type move thaw**: Being hit by a damaging Fire-type move thaws the target
- **Self-thaw moves**: Flame Wheel, Sacred Fire thaw the USER when used while frozen
- **Wake turn**: Pokémon CAN act on the turn it thaws (can use a move)
- **Application**: Ice-type Pokémon cannot be frozen (confirmed for Gen 3)
- **Sun interaction**: Cannot be frozen while Sun is active (confirmed Gen 3+)

### Volatile Status (Lost on switch-out)

#### Confusion
- **Self-hit chance**: 50% per turn
- **Self-hit damage**: Calculated as a typeless 40-power physical move against self (uses own Attack vs own Defense)
- **Duration**: 2–5 turns (inclusive)
- **Interactions**: Own Tempo ability prevents confusion

#### Infatuation (Attract)
- **Fail chance**: 50% per turn to be unable to act ("immobilized by love")
- **Conditions**: Only works on opposite gender. Genderless Pokémon are immune.
- **Duration**: Until the infatuating Pokémon switches out or faints
- **Oblivious ability**: Immune to Attract

#### Leech Seed
- **Drain**: 1/8 max HP per turn, heals the Pokémon that planted it
- **Liquid Ooze interaction**: Draining Pokémon takes damage instead of healing
- **Grass-type**: Immune (Leech Seed fails on Grass-type targets)
- **Duration**: Persists until switch-out

#### Taunt
- **Duration**: 2 turns in Gen 3 (changed to 3 in Gen 4+)
- **Effect**: Target can only use damaging moves (status moves fail)

#### Encore
- **Duration**: 2–5 turns in Gen 3
<!-- VERIFY_POKEEMERALD: Encore duration. Some sources say 3-8 for Gen 3. -->
- **Effect**: Target is forced to repeat the last move it used

#### Torment
- **Effect**: Target cannot use the same move on consecutive turns
- **Duration**: Until switch-out

#### Curse (Ghost-type user)
- **User cost**: Loses 50% of own max HP
- **Effect on target**: Takes 1/4 max HP damage per turn
- **Duration**: Until switch-out

#### Disable
- **Duration**: 2–5 turns in Gen 3
- **Effect**: Disables the last move the target used

## 11. Accuracy & Evasion

### Accuracy Check Formula
```typescript
hitChance = floor(moveAccuracy * accuracyStageMultiplier / evasionStageMultiplier)
// Random integer 1-100. If random ≤ hitChance, move hits.
```

Moves with `accuracy: null` or `accuracy: true` bypass the accuracy check entirely (Swift, Aerial Ace, Shock Wave, Faint Attack, Vital Throw, Magical Leaf).

### Stat Stage Multiplier Table

Accuracy and Evasion stages use the same table but in opposite directions:

| -3 | 3/6 | 0.500 |
| -2 | 3/5 | 0.600 |
| -1 | 3/4 | 0.750 |

| +1 | 4/3 | 1.333 |
| +2 | 5/3 | 1.667 |
| +3 | 6/3 | 2.000 |
| +4 | 7/3 | 2.333 |
| +5 | 8/3 | 2.667 |
| +6 | 9/3 | 3.000 |

### OHKO Move Accuracy
- Formula: `(attacker's level - target's level + 30)%`
- Minimum: 30% (when levels are equal)
- Maximum: 100%
- Fails automatically if attacker's level < target's level
- Not affected by accuracy/evasion modifiers
- Blocked by Sturdy ability

## 13. Protect & Detect

### Protect / Detect
- **Priority**: +3
- **Effect**: Prevents all damage and most status moves for one turn
- **Successive use**: Success rate halves each consecutive turn:
  - 1st: ~100% (see bug note)
  - 2nd: ~50%
  - 3rd: ~25%
  - 4th: ~12.5%
  - Further: continues halving
- **Ruby/Sapphire bug**: 1/65,536 chance of failing on first use (wrong comparison operator `>=` instead of `>`). Emerald fixed this.
- **Ruby/Sapphire bug**: Consecutive use counter reads garbage memory after 5+ uses. Emerald has the correct implementation.
- **Using a different move resets the counter**
- **Protect and Detect share the same counter** (alternating doesn't help)
- **Feint**: Does NOT exist in Gen 3 (Gen 4). No move breaks Protect in Gen 3.

## 18. Implementation Notes

### BaseRuleset Structure

Gen 3 is the **base generation** for `BaseRuleset`. The abstract class implements all Gen 3 defaults. Later generations (4-9) extend `BaseRuleset` and override only what changed.

```typescript
// Gen 3 extends BaseRuleset (which provides Gen 3 defaults)
class Gen3Ruleset extends BaseRuleset {
  readonly generation = 3

  // Override specific methods if Gen 3 has quirks that differ from
  // the "generic Gen 3+" implementation in BaseRuleset.
  // Most Gen 3 behavior IS the BaseRuleset default.
}
```

### Key Implementation Principles

1. **Physical/Special by type**: Determine category from move type, NOT from move metadata. This is the single most important Gen 3 distinction from Gen 4+.

2. **Integer truncation everywhere**: Every division in damage/stat calculation uses `Math.floor()`. Never round.

3. **Type effectiveness sequentially**: For dual-typed defenders, apply each type's multiplier separately with `floor()` between.

4. **Weather abilities are permanent**: Drizzle/Drought/Sand Stream weather does NOT have a turn counter. Only move-based weather counts down.

5. **No sandstorm SpDef boost**: This is a Gen 4+ mechanic. Do NOT implement it for Gen 3.

6. **Abilities are mandatory**: Every Pokémon has exactly 1 ability. No hidden ability slot.

7. **Sitrus Berry is 30 HP flat**: Not percentage-based. This changed in Gen 4.

### What Gen 3 Overrides in BaseRuleset

If `BaseRuleset` is designed around the "most common" Gen 3+ behavior:
- Most methods should be the direct Gen 3 implementation (no override needed)
- Override candidates: only quirks specific to RSE/FRLG that were changed in Gen 4
