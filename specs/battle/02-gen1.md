## 1. Overview

Generation 1 (Red/Blue/Yellow) introduced the foundational battle system that would persist through later generations, but with several critical differences that make Gen 1 mechanically distinct:

- **No Abilities**: Pokémon have no abilities whatsoever. This drastically simplifies battle mechanics.
- **No Held Items**: Items exist in the game but cannot be held during battle.
- **Unified Special Stat**: A single "Special" stat governs both Special Attack and Special Defense. Later gens split this into SpA and SpD.
- **No Physical/Special Split Per Move**: The categorization of moves as physical or special is entirely type-based. All Normal-type moves are physical, all Grass-type moves are special, etc.
- **Type-Based Damage Scaling**: Physical moves use Attack/Defense, Special moves use Special/Special.
- **Simplified Status System**: No permanent stat modifications carry between battles; simpler condition mechanics.
- **Critical Hit Based on Speed Stat**: Crit probability is determined by base Speed, not by held items or move properties.
- **No Priority System** (mostly): Except for Counter (−1) and Quick Attack (+1), all moves resolve in Speed order.
- **No Weather, Terrain, or Hazards**: The battle environment is static.
- **Multiple Critical Bugs**: The 1/256 miss bug, Ghost→Psychic immunity bug, Focus Energy bug, badge boost glitch, stat overflow, toxic counter bug, and others are all present.

This is a fundamentally different system from Gen 3+ and requires its own complete implementation.

## 2. Stat System

Gen 1 uses DVs (Determinant Values) and Stat Experience, not the modern IV/EV system.

### DVs and Stat Experience

- **DVs (Determinant Values)**: 4-bit values, 0–15 per stat. *Not* called IVs (that term is Gen 3+).
- **Stat Experience**: 0–65535 per stat. No per-stat cap; all five stats can be maxed independently.
  - Gained by defeating Pokémon: you gain stat exp equal to the defeated Pokémon's base stats.
  - **Bonus calculation**: `statExpBonus = floor(ceil(sqrt(statExp)) / 4)`
  - Maximum bonus: `ceil(sqrt(65535)) = 256` → `floor(256 / 4) = 64` at level 100.

### HP DV Derivation

The HP DV is **not independently chosen** — it is derived from the parity of the other four DVs:

```
HP_DV = (Attack_DV & 1) << 3 | (Defense_DV & 1) << 2 | (Speed_DV & 1) << 1 | (Special_DV & 1)
```

Result is 0–15 based on whether the four other DVs are odd or even.

### Stat Formula

```typescript
interface Gen1Stats {
  hp: number;
  attack: number;
  defense: number;
  special: number;  // Unified Special stat (later split into SpA/SpD)
  speed: number;
}

function calculateGen1Stat(
  baseValue: number,
  dv: number,      // Determinant Value: 0-15
  statExp: number, // Stat Experience: 0-65535
  level: number
): number {
  const statExpBonus = Math.floor(Math.ceil(Math.sqrt(statExp)) / 4);
  return Math.floor(((baseValue + dv) * 2 + statExpBonus) * level / 100) + 5;
}

function calculateGen1HP(
  baseHP: number,
  dv: number,      // HP DV: derived from other DVs (see above)
  statExp: number, // Stat Experience: 0-65535
  level: number
): number {
  const statExpBonus = Math.floor(Math.ceil(Math.sqrt(statExp)) / 4);
  return Math.floor(((baseHP + dv) * 2 + statExpBonus) * level / 100) + level + 10;
}
```

### Stat Stage Multipliers

Stat stages use a numerator/denominator system: `numerator = max(2, 2 + stage)`, `denominator = max(2, 2 - stage)`.

Applied as `Math.floor(stat * numerator / denominator)`.

| Stage | Multiplier |
|-------|-----------|
| −6    | 2/8 = 25% |
| −5    | 2/7 ≈ 28.6% |
| −4    | 2/6 ≈ 33.3% |
| −3    | 2/5 = 40% |
| −2    | 2/4 = 50% |
| −1    | 2/3 ≈ 66.7% |
| 0     | 2/2 = 100% |
| +1    | 3/2 = 150% |
| +2    | 4/2 = 200% |
| +3    | 5/2 = 250% |
| +4    | 6/2 = 300% |
| +5    | 7/2 = 350% |
| +6    | 8/2 = 400% |

```typescript
function applyStatStage(baseStat: number, stage: number): number {
  const numerator = Math.max(2, 2 + stage);
  const denominator = Math.max(2, 2 - stage);
  return Math.floor(baseStat * numerator / denominator);
}
```

### Special as Unified Stat

The "Special" stat affects both:
- **Damage Output**: Special moves use Special as their attack stat
- **Damage Mitigation**: Special moves use Special as the defense stat

This means a Pokémon with low Special takes more damage from special moves AND deals less Special damage.

### Badge Boosts

| Badge   | Stat    |
|---------|---------|
| Boulder | Attack  |
| Thunder | Defense |
| Soul    | Speed   |
| Volcano | Special |

Multiplier: 9/8 (1.125×), floored. **Not applied in link battles.**

**Badge boost glitch**: When any stat stage is modified in battle, badge boosts are re-applied to the modified stat, causing compounding. Stats can stack up to 999.

## 4. Damage Formula

The Gen 1 damage formula is fundamentally different from Gen 3+ formulas. It uses integer division at multiple points and applies modifiers sequentially (not as a combined float multiplier).

> **Important (Gen 1 specific)**: Type effectiveness against dual-typed Pokémon must be applied **sequentially with integer floor between each type**, not as a combined multiplier. This matches the actual Gen 1 implementation.
>
> Example: Fire vs. Water/Rock
> - Step 1: fire vs water = 0.5; damage_intermediate = floor(damage * 0.5)
> - Step 2: fire vs rock = 0.5; final_damage = floor(damage_intermediate * 0.5)
>
> This differs from the combined multiplier approach (0.25) due to floor behavior on odd numbers.
>
> **Code bug note**: `Gen1DamageCalc.ts` ~line 159 uses combined float multiplier — this is a known bug flagged for separate fix.

### Physical vs Special Categories

Move category is determined **entirely by type** in Gen 1:

```typescript
function getMoveCategoryByType(moveType: string): "physical" | "special" {
  const physicalTypes = [
    "normal", "fighting", "flying", "ground", "rock", "bug", "ghost", "poison"
    // NOTE: Poison is Physical in Gen 1
  ];

  const specialTypes = [
    "fire", "water", "electric", "grass", "ice", "psychic", "dragon"
    // NOTE: Dragon is Special in Gen 1
  ];

  return physicalTypes.includes(moveType) ? "physical" : "special";
}
```

### Complete Gen 1 Damage Formula

```typescript
function calculateGen1Damage(
  level: number,            // Level of attacking Pokémon
  attack: number,           // Effective Attack or Special (after stat stages, burn halving)
  defense: number,          // Effective Defense or Special (after stat stages, Reflect/Light Screen)
  basePower: number,        // Base Power of the move
  isCritical: boolean,      // If true: use unmodified stats (no stages, no screens)
  moveType: string,         // Move's type
  attackerTypes: string[],  // Attacker's types (for STAB)
  defenderType1: string,    // Defender's primary type
  defenderType2: string | null, // Defender's secondary type
  random: number            // Integer 217-255 (inclusive)
): number {

  // --- Step 1: Effective level (doubled for crits)
  const effectiveLevel = isCritical ? level * 2 : level;

  // --- Step 2: Stat selection
  // If crit: ignore stat stages, Reflect/Light Screen, burn halving
  // (caller passes unmodified stats in that case)

  // --- Step 3: Stat overflow handling
  // BUG: If Attack OR Defense >= 256, both are divided by 4 mod 256
  // This can cause a division-by-zero crash if Defense becomes 0
  let effectiveAttack = attack;
  let effectiveDefense = defense;
  if (effectiveAttack >= 256 || effectiveDefense >= 256) {
    effectiveAttack = (Math.floor(effectiveAttack / 4)) % 256;
    effectiveDefense = (Math.floor(effectiveDefense / 4)) % 256;
    if (effectiveAttack === 0) effectiveAttack = 1;
    // If effectiveDefense === 0: division by zero — original game crashes
    // In emulation, handle gracefully (e.g., set to 1 or return 0)
  }

  // --- Step 4: Base damage calculation
  let damage = Math.floor(
    Math.floor(
      Math.floor(effectiveLevel * 2 / 5 + 2) * basePower * effectiveAttack / effectiveDefense
    ) / 50
  ) + 2;

  // --- Step 5: Cap base damage at 997
  damage = Math.min(damage, 997);

  // --- Step 6: STAB (Same-Type Attack Bonus)
  // NOT a 1.5× multiplier — it's: damage += floor(damage / 2)
  if (attackerTypes.includes(moveType)) {
    damage = damage + Math.floor(damage / 2);
  }

  // --- Step 7: Type effectiveness — applied sequentially, with integer division each step
  // 2x: floor(damage * 20 / 10)
  // 0.5x: floor(damage * 5 / 10)
  // 0x: damage = 0
  damage = applyTypeEffectiveness(damage, moveType, defenderType1);
  if (defenderType2 !== null) {
    damage = applyTypeEffectiveness(damage, moveType, defenderType2);
  }

  // --- Step 8: Minimum damage before random (if non-immune)
  if (damage === 0 && !isImmune(moveType, defenderType1, defenderType2)) {
    damage = 1;
  }

  // --- Step 9: Random factor
  // 39 discrete values: 217, 218, ..., 255
  // floor(damage * random / 255)
  damage = Math.floor(damage * random / 255);

  // --- Step 10: Minimum final damage (if non-immune)
  if (damage === 0 && !isImmune(moveType, defenderType1, defenderType2)) {
    damage = 1;
  }

  return damage;
}

function applyTypeEffectiveness(damage: number, moveType: string, defenderType: string): number {
  const mult = getSingleTypeMultiplier(moveType, defenderType);
  if (mult === 2) {
    return Math.floor(damage * 20 / 10);
  } else if (mult === 0.5) {
    return Math.floor(damage * 5 / 10);
  } else if (mult === 0) {
    return 0;
  }
  return damage; // 1x: no change
}

function isImmune(moveType: string, type1: string, type2: string | null): boolean {
  return getSingleTypeMultiplier(moveType, type1) === 0 ||
    (type2 !== null && getSingleTypeMultiplier(moveType, type2) === 0);
}

function getSingleTypeMultiplier(moveType: string, defenderType: string): number {
  // Immunities
  if ((moveType === "normal" && defenderType === "ghost") ||
      (moveType === "ghost" && defenderType === "normal") ||
      (moveType === "ghost" && defenderType === "psychic") || // BUG: should be 2x
      (moveType === "electric" && defenderType === "ground") ||
      (moveType === "ground" && defenderType === "flying")) {
    return 0;
  }

  const superEffective: Record<string, string[]> = {
    "fire":     ["bug", "grass", "ice"],
    "water":    ["fire", "ground", "rock"],
    "electric": ["water", "flying"],
    "grass":    ["water", "ground", "rock"],
    "ice":      ["flying", "ground", "grass"],
    "fighting": ["normal", "ice", "rock"],
    "poison":   ["grass", "bug"],
    "ground":   ["fire", "electric", "poison", "rock"],
    "flying":   ["bug", "grass", "fighting"],
    "psychic":  ["fighting", "poison"],
    "bug":      ["grass", "psychic", "poison"],
    "rock":     ["bug", "fire", "flying", "ice"],
    "ghost":    ["ghost"],
  };

  if (superEffective[moveType]?.includes(defenderType)) {
    return 2;
  }

  const notVeryEffective: Record<string, string[]> = {
    "fire":     ["water", "ground", "rock"],
    "water":    ["electric", "grass"],
    "grass":    ["fire", "ice", "poison", "flying", "bug"],
    "ice":      ["fire", "fighting", "rock"],
    "fighting": ["flying", "psychic"],
    "poison":   ["ground", "psychic"],
    "ground":   ["water", "grass", "ice"],
    "flying":   ["electric", "ice", "rock"],
    "psychic":  ["bug", "ghost"],
    "bug":      ["fire", "flying", "rock"],
    "rock":     ["water", "grass", "fighting", "ground"],
    "ghost":    ["poison", "bug"],
  };

  if (notVeryEffective[moveType]?.includes(defenderType)) {
    return 0.5;
  }

  return 1;
}
```

### Random Factor

The random factor is an **integer** in the range 217–255 inclusive (39 discrete values):
- 217/255 ≈ 85.1% (minimum damage)
- 255/255 = 100% (maximum damage)

Applied as: `Math.floor(damage * random / 255)`

This is distinct from later gens which use a float multiplier.

### Self-Destruct and Explosion

Self-Destruct and Explosion **halve the target's Defense** during the damage calculation (before the division step). This effectively doubles their power. The user always faints, even on miss or if the target is immune.

```typescript
// BUG/MECHANIC: Target's Defense is halved before damage calc for Self-Destruct/Explosion
if (move.name === "Self-Destruct" || move.name === "Explosion") {
  defense = Math.floor(defense / 2);
}
```

## 5. Critical Hits

Critical hit mechanics in Gen 1 are radically different from later generations.

### Critical Hit Rate Calculation

```typescript
function calculateCriticalHitThreshold(
  attackerBaseSpeed: number,
  isHighCritMove: boolean,  // Slash, Razor Leaf, Crabhammer, Karate Chop
  focusEnergyActive: boolean
): number {
  // Base threshold
  let threshold: number;
  if (isHighCritMove) {
    // High-crit moves have 8× the normal crit rate
    threshold = Math.floor(attackerBaseSpeed * 8 / 2);
  } else {
    threshold = Math.floor(attackerBaseSpeed / 2);
  }

  // Cap at 255
  threshold = Math.min(threshold, 255);

  // Focus Energy bug: divides by 4 instead of multiplying by 4
  if (focusEnergyActive) {
    // BUG: Intended to multiply by 4; actually divides by 4
    threshold = Math.floor(threshold / 4);
  }

  return threshold;
}

// Critical hit occurs if: random(0..255) < threshold
// Probability = threshold / 256
```

### High-Crit Moves

These moves have 8× the crit rate: **Slash, Razor Leaf, Crabhammer, Karate Chop**.

Formula: `threshold = floor(baseSpeed * 8 / 2)` capped at 255.

For base Speed ≥ 32 (threshold reaches 128+), high-crit moves land crits more often than not.

### Crit Rate Examples

| Pokémon    | Base Speed | Normal Crit Rate      | High-Crit Rate              |
|------------|-----------|----------------------|----------------------------|
| Alakazam   | 120       | 60/256 ≈ 23.4%       | 255/256 ≈ 99.6%            |
| Tauros     | 110       | 55/256 ≈ 21.5%       | 255/256 ≈ 99.6%            |
| Chansey    | 50        | 25/256 ≈ 9.8%        | 200/256 ≈ 78.1%            |
| Snorlax    | 30        | 15/256 ≈ 5.9%        | 120/256 ≈ 46.9%            |
| Mewtwo     | 130       | 65/256 ≈ 25.4%       | 255/256 ≈ 99.6%            |

Note: Normal crit rate **never reaches 100%** for any base Speed. Speed 255 gives 127/256 ≈ 49.6%.

### Focus Energy (Crit Rate Calculation — Gen 1 Bug)

Focus Energy was intended to quadruple the critical hit rate but due to a bug, it **halves** it instead.

The Showdown-accurate algorithm for Gen 1 crit threshold:
1. Start with `baseCritThreshold = floor(baseSpeed / 2)`
2. If the move is a high-crit move: multiply by 4 (capped at 255)
3. If using a normal move (not high-crit): divide by 2 (floor)
4. If Focus Energy is active: divide by 2 again (floor) — this is the bug!

So for a normal move with Focus Energy: `floor(floor(baseSpeed / 2) / 2) / 2 = floor(baseSpeed / 8)`

For a high-crit move with Focus Energy: `floor(floor(baseSpeed / 2) * 4 / 2) = floor(baseSpeed)` (capped at 255)

The crit check: `rng.int(0, 255) < threshold` → critical hit.

Focus Energy (and Dire Hit) divide the crit threshold by 4 instead of multiplying by 4. Makes crits **less** likely, not more.

```typescript
// Intended: critRate *= 4
// Actual Gen 1: critRate = Math.floor(critRate / 4)
```

### Critical Hit Damage

Critical hits **double the effective level** in the damage formula (not a ×2 damage multiplier):

```typescript
// Non-crit: effectiveLevel = level
// Crit:     effectiveLevel = level * 2
const effectiveLevel = isCritical ? level * 2 : level;
```

Additionally, critical hits **ignore ALL stat stages** on both attacker and defender, and **ignore Reflect and Light Screen**. The unmodified base stats are used.

## 6. Status Conditions

Gen 1 status conditions are simpler than later gens but have unique quirks.

### Status Types

```typescript
type Gen1PrimaryStatus = "none" | "burn" | "freeze" | "paralysis" | "poison" | "sleep";
// Note: "badly-poisoned" (toxic) is represented as poison with an escalating counter

interface StatusState {
  condition: Gen1PrimaryStatus;
  counter: number;  // Sleep: turns remaining; Toxic: damage multiplier (1, 2, 3...)
}
```

### Burn

- **Attack reduced to 50%**: `Math.floor(attackStat / 2)` — applied to the stat value directly, not as a stage
- **1/16 max HP damage per turn** at end of turn
- Permanent until cured by item or Haze

```typescript
function applyBurnAtk(baseStat: number): number {
  return Math.floor(baseStat / 2);  // Not a stage modifier
}

function applyBurnEndOfTurn(pokemon: Pokemon): void {
  const damage = Math.max(1, Math.floor(pokemon.maxHP / 16));
  pokemon.currentHP -= damage;
}
```

**Toxic counter bug**: Burn damage uses the **same internal counter** as Toxic. If a Pokémon was previously afflicted with Toxic (then cured), then burned, the burn damage escalates using the leftover Toxic counter value.

### Freeze

- Pokémon cannot attack — fully immobilized
- **No natural thaw chance** (0% per turn) — unlike later gens
- Thaw conditions:
  1. Hit by a **damaging Fire-type move**
  2. **Haze** (unique to Gen 1 — cures all statuses)
  3. Items (Ice Heal, Full Restore, etc.)

```typescript
// BUG: There is no "20% thaw on each turn" mechanic in Gen 1
// A frozen Pokémon will never self-thaw
```

### Paralysis

- **25% chance of full paralysis** (63/256) each turn: Pokémon cannot act
- **Speed quartered**: `Math.floor(speedStat * 0.25)` — applied to stat value immediately

```typescript
function applyParalysisSpeed(speedStat: number): number {
  return Math.floor(speedStat * 0.25);
}
```

### Sleep

- Duration: 1–7 turns (random counter set on application)
- **Counter decremented before action check**: when counter hits 0, Pokémon wakes but **forfeits its action that turn**
- A Pokémon cannot act on the turn it wakes up (effectively +1 sleep turn)
- Sleep counter does **not** reset on switch-out — it persists

```typescript
function handleSleepTurn(pokemon: Pokemon): "asleep" | "just-woke" {
  // Counter is decremented FIRST
  pokemon.status.counter -= 1;

  if (pokemon.status.counter <= 0) {
    // Pokémon wakes up — but CANNOT act this turn
    pokemon.status.condition = "none";
    pokemon.status.counter = 0;
    return "just-woke"; // signal to skip turn
  }
  return "asleep";
}
```

### Poison (Regular)

- **1/16 max HP damage per turn** at end of turn
- Permanent until cured

```typescript
function applyPoisonEndOfTurn(pokemon: Pokemon): void {
  const damage = Math.max(1, Math.floor(pokemon.maxHP / 16));
  pokemon.currentHP -= damage;
}
```

### Toxic (Badly Poisoned)

- Starts at 1/16 max HP damage, escalates by 1/16 each turn: 1/16, 2/16, 3/16, ...
- Counter (`N`) increments each turn; damage = `floor(maxHP * N / 16)`
- Counter persists through switch-out in Gen 1 (unlike Gen 2+ where it resets to 1)
- **Shared counter bug**: burn, poison, and Leech Seed all share this counter (see Burn section)

```typescript
function applyToxicEndOfTurn(pokemon: Pokemon): void {
  const n = pokemon.status.counter;
  const damage = Math.max(1, Math.floor(pokemon.maxHP * n / 16));
  pokemon.currentHP -= damage;
  pokemon.status.counter += 1;
}
```

### Confusion

- Duration: 2–5 turns (random); counter decremented before check. pokered `effects.asm:1143-1147`: `and $3; inc a; inc a` = random(0-3)+2 = 2-5.
- 50% chance to hit self each turn
- Self-hit: 40 power, typeless, physical — uses attacker's own Attack and Defense stats
- Clears on switch-out

**Confusion/Substitute bug**: If a confused Pokémon behind a Substitute hits itself, the damage is applied to the **opponent's Substitute** instead.

## 7. Move Mechanics

### Recharge Moves (Hyper Beam)

Hyper Beam requires a recharge turn after use. However, recharge is **skipped** in the following cases:

```typescript
function requiresRecharge(
  move: Move,
  targetFainted: boolean,
  brokeSub: boolean,
  moveMissed: boolean
): boolean {
  if (!move.requiresRecharge) return false;

  // Hyper Beam skips recharge if:
  if (targetFainted) return false;   // KO'd the target
  if (brokeSub) return false;        // Broke a Substitute
  if (moveMissed) return false;      // Missed

  return true; // Otherwise must recharge
}
```

### Counter

Counter is a move with −1 priority that deals 2× the damage received on the previous turn.

**Important restriction**: Counter only reflects damage from **Normal-type and Fighting-type moves** — not all physical moves.

```typescript
function resolveCounter(
  opponentLastMoveType: string,
  opponentLastDamage: number
): number {
  // Counter ONLY reflects Normal-type and Fighting-type move damage
  // NOT all physical moves
  if (opponentLastMoveType === "normal" || opponentLastMoveType === "fighting") {
    return opponentLastDamage * 2;
  }
  return 0; // Counter fails against all other types
}
```

### Struggle

Used when all moves are out of PP.

- 50 Base Power, Normal type
- Subject to the 1/256 miss bug (despite being "unavoidable")
- **Recoil: 50% of damage dealt** (not a percentage of max HP)

```typescript
function applyStruggleRecoil(pokemon: Pokemon, damageDealt: number): void {
```

### Substitute

## 8. Accuracy & Evasion

### Accuracy Check Formula

```typescript
function resolveAccuracyCheck(
  moveBaseAccuracy: number, // e.g., 100 for "100% accuracy"
  attackerAccuracyStage: number, // -6 to +6
  defenderEvasionStage: number   // -6 to +6
): boolean {
  const accNumerator = Math.max(2, 2 + attackerAccuracyStage);
  const accDenominator = Math.max(2, 2 - attackerAccuracyStage);
  const evaNumerator = Math.max(2, 2 + defenderEvasionStage);
  const evaDenominator = Math.max(2, 2 - defenderEvasionStage);

  // Accuracy stage boosts accuracy; evasion stage reduces it
  let threshold = Math.floor(
    moveBaseAccuracy * accNumerator / accDenominator * evaDenominator / evaNumerator
  );

  // Clamp to 1-255
  threshold = Math.min(255, Math.max(1, threshold));

  const roll = Math.floor(Math.random() * 256); // 0-255
  return roll < threshold;
}
```

### Accuracy / Evasion Stage Multipliers

Uses the same numerator/denominator system as stat stages.
`numerator = max(2, 2 + stage)`, `denominator = max(2, 2 - stage)`.

Accuracy: multiply by `numerator / denominator`.
Evasion: multiply by `denominator / numerator` (inverted — higher evasion = harder to hit).

### The 1/256 Miss Glitch

Moves with 100% base accuracy are stored as 255 internally (0xFF). The hit check is `random(0..255) < threshold`. When threshold = 255, the roll of 255 still misses:

```typescript
// If a 100% accuracy move has no stage modifiers: threshold = 255
// random(0..255) < 255 fails when roll = 255 (1/256 chance)
// This affects ALL moves with 100% accuracy except Swift and self-targeting moves
```

**Swift** bypasses the accuracy check entirely. Self-targeting moves also bypass the 1/256 miss glitch — their accuracy threshold is set to 256 (instead of 255), so the roll can never equal or exceed it.

### OHKO Moves (Horn Drill, Guillotine, Fissure)

- Fail automatically if user's Speed < target's Speed
- Base accuracy: 30%
- X Accuracy makes them auto-hit (bypasses accuracy check, but Speed comparison still applies)

## 9. Turn Order

### Speed-Based Turn Order

```typescript
function resolveTurnOrder(
  pokemon1: Pokemon, move1: Move,
  pokemon2: Pokemon, move2: Move
): [Pokemon, Move, Pokemon, Move] {
  const priority1 = getMovePriority(move1);
  const priority2 = getMovePriority(move2);

  if (priority1 !== priority2) {
    // Higher priority goes first
    return priority1 > priority2
      ? [pokemon1, move1, pokemon2, move2]
      : [pokemon2, move2, pokemon1, move1];
  }

  // Same priority: higher Speed goes first
  const speed1 = getEffectiveSpeed(pokemon1);
  const speed2 = getEffectiveSpeed(pokemon2);

  if (speed1 !== speed2) {
    return speed1 > speed2
      ? [pokemon1, move1, pokemon2, move2]
      : [pokemon2, move2, pokemon1, move1];
  }

  // Speed tie: random 50/50
  return Math.random() < 0.5
    ? [pokemon1, move1, pokemon2, move2]
    : [pokemon2, move2, pokemon1, move1];
}

function getMovePriority(move: Move): number {
  if (move.name === "Quick Attack") return 1;
  if (move.name === "Counter") return -1;
  return 0;
}

function getEffectiveSpeed(pokemon: Pokemon): number {
  let speed = pokemon.stats.speed;
  speed = applyStatStage(speed, pokemon.statStages.speed);
  if (pokemon.status.condition === "paralysis") {
    speed = Math.floor(speed * 0.25);
  }
  return speed;
}
```

### Switches

Switches always execute before moves — a switching Pokémon's switch resolves before any move is used that turn.

## 10. End-of-Turn Effects

### End-of-Turn Event Order

1. **Burn damage** — 1/16 max HP
2. **Poison damage** — 1/16 max HP (or N/16 for Toxic, escalating)
3. **Leech Seed drain** — 1/16 max HP
4. **Faint check** — if HP ≤ 0, Pokémon faints

Note: The toxic counter bug means burn, poison, and Leech Seed share a damage counter. Burn can escalate if a Toxic counter was accumulated previously.

```typescript
function resolveEndOfTurn(pokemon: Pokemon): void {
  if (pokemon.status.condition === "burn") {
    const damage = Math.max(1, Math.floor(pokemon.maxHP / 16));
    pokemon.currentHP -= damage;
    // Note: subject to toxic counter bug — may escalate if counter > 1
  }

  if (pokemon.status.condition === "poison") {
    const damage = Math.max(1, Math.floor(pokemon.maxHP / 16));
    pokemon.currentHP -= damage;
  }

  // Toxic: escalating damage
  if (pokemon.status.condition === "toxic") {
    const n = pokemon.status.counter;
    const damage = Math.max(1, Math.floor(pokemon.maxHP * n / 16));
    pokemon.currentHP -= damage;
    pokemon.status.counter += 1;
  }

  // Leech Seed (if active): 1/16 max HP drain, heals opponent
  if (pokemon.leechSeedActive) {
    const drain = Math.max(1, Math.floor(pokemon.maxHP / 16));
    pokemon.currentHP -= drain;
    // opponent.currentHP = Math.min(opponent.maxHP, opponent.currentHP + drain);
  }
}
```

### Reflect and Light Screen Duration

Reflect and Light Screen have **no turn limit**. They remain active until the Pokémon switches out or faints. There is no 5-turn countdown in Gen 1.

### What Resets on Switch-Out

- All stat stages (reset to 0)
- Confusion
- Toxic counter (damage reverts to base 1/16 rate)
- Disable
- Rage state
- Type changes from Conversion
- Trapping/binding effect
- Focus Energy
- Reflect/Light Screen

### What Persists on Switch-Out

- Sleep counter (does NOT reset — persists until naturally expired)
- Primary status condition (burn, freeze, paralysis, poison, sleep)

## 11. Notable Bugs & Quirks

### 1. Ghost → Psychic Immunity Bug

**Description**: Ghost-type moves have 0× effectiveness against Psychic types (immune), when they should be super-effective (2×). The type chart entry was coded incorrectly.

**Effect**: Psychic types have **no real weakness** in Gen 1. Bug type moves exist but are rare and weak; Ghost type can't even hit Psychic types.

```typescript
// BUG: Ghost → Psychic = 0x (coded as immune)
// INTENDED: Ghost → Psychic = 2x (super-effective)
```

### 2. Focus Energy Bug

Focus Energy (and Dire Hit) divide the crit threshold by 4 instead of multiplying by 4. Makes crits **less** likely, not more.

```typescript
// BUG: critThreshold = floor(critThreshold / 4)
// INTENDED: critThreshold = critThreshold * 4
```

### 3. 1/256 Miss Glitch

All moves with 100% accuracy have a 1/256 chance to miss. Swift is the only exception — it bypasses the roll entirely.

### 4. Badge Boost Glitch

When a stat stage is modified in battle, the player's badge boosts are re-applied to the modified stat, causing compounding. This can stack stats up to 999.

### 5. Toxic Counter Bug

Burn damage, regular poison damage, and Leech Seed all use the same internal damage counter as Toxic. If a Pokémon was previously Toxic'd (then cured), then burned, the burn damage will escalate as if it were escalating Toxic.

### 6. Stat Overflow

If Attack or Defense reaches 256+ during damage calculation (from stat boosts), **both** values are divided by 4 mod 256:

```typescript
// BUG: stat overflow in damage calculation
if (attack >= 256 || defense >= 256) {
  attack = Math.floor(attack / 4) % 256;
  defense = Math.floor(defense / 4) % 256;
  if (attack === 0) attack = 1;
  // defense === 0 causes division-by-zero crash in original game
}
```

This can result in a devastating reduction in damage — or a game crash if Defense wraps to 0.

### 7. Confusion/Substitute Bug

If a confused Pokémon behind a Substitute hits itself in confusion, the self-hit damage is applied to the **opponent's Substitute** instead of the user.

### 8. Hyper Beam No-Recharge

Hyper Beam skips its recharge turn if it KOs the target, breaks a Substitute, or misses (see Section 7).

### 9. Critical Hits Ignore Stat Stages

Critical hits recalculate damage using unmodified base stats — ignoring all stat stage boosts and drops on both the attacker and defender, and ignoring Reflect/Light Screen.

### 10. Wrap and Trapping Moves Overpowered

Trapping moves prevent the opponent from switching or attacking, can be reused indefinitely, and allow the faster Pokémon to trap-lock the opponent for the entire battle.

### 11. Sleep: No Action on Wake Turn

A Pokémon cannot act on the turn it wakes from sleep. The sleep counter is decremented first; when it reaches 0, the Pokémon wakes but forfeits its turn.

### 12. Rage Miss Loop

If Rage misses due to accuracy/evasion, it can get stuck in a permanent miss loop. The Pokémon is effectively unable to act for the rest of the battle.

### Key Implementation Decisions

1. **Stat formula uses DV (0–15) and Stat Experience (0–65535) with sqrt**, not the modern IV/EV formula. Completely separate implementation from Gen 3+.

2. **HP constant is +10, not +5**. Easy to get wrong from memory.

3. **HP DV is derived**, not independently stored. Compute it from the other four DVs.

4. **Type effectiveness is sequential integer division**, not a combined float multiplier. Two 2× hits = `floor(floor(damage * 20/10) * 20/10)`, which differs from `damage * 4` after rounding.

5. **STAB is `damage += floor(damage / 2)`**, not `damage *= 1.5`. Subtle difference with integer division.

6. **Critical hits double the level**, not the damage. Crits use unmodified stats and ignore screens.

7. **Poison and Burn both deal 1/16 max HP per turn**. The old "regular poison does no damage" claim was wrong.

8. **Reflect and Light Screen are permanent** (no 5-turn limit). Reset only on switch-out or faint.

9. **Counter only reflects Normal and Fighting moves**, not all physical moves.

10. **Stat overflow** (Attack or Defense ≥ 256) must be handled explicitly — both values get divided by 4 mod 256.

11. **No abilities, no held items, no weather, no terrain**. Remove all such checks from the Gen1 implementation.

### Testing Considerations

1. **Stat formula**: Test that level 100 Chansey (HP 250 base, max DV 15, max Stat Exp 65535) gives the correct HP value using the DV/sqrt formula.

2. **HP DV derivation**: Test that HP DV is correctly computed from the other four DVs.

3. **Damage formula**: Test STAB + type-effectiveness integer division produces correct values. Verify the 997 base damage cap.

4. **Crit rate examples**: Verify Alakazam (120 base Speed) gets 60/256 ≈ 23.4% with normal moves, and ~99.6% with Slash (high-crit).

5. **Status damage**: Both regular poison and burn should deal `Math.max(1, Math.floor(maxHP / 16))` per turn.

6. **Reflect/Light Screen**: Verify they do not expire after 5 turns — they should persist until switch-out.

7. **Counter type restriction**: Verify Counter returns 0 for Rock-type or Ground-type physical moves.

8. **Ghost immunity**: Verify Ghost → Psychic returns 0 (not 1 or 2).

9. **Poison type**: Verify Poison-type moves use Attack/Defense (physical), not Special.

10. **Sleep wake**: Verify the Pokémon cannot act on the turn it wakes from sleep.
