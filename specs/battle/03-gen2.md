# Pokémon Gen 2 (Gold/Silver/Crystal) Battle Mechanics Specification

> Types and interfaces reference `@pokemon-lib-ts/core` — see `core/01-entities.md` for canonical definitions.

## 1. Overview

Gen 2 introduced fundamental changes to the core mechanics established in Gen 1, most notably the **Special stat split** into Special Attack and Special Defense, along with a suite of new systems that expanded strategic depth.

### Major Changes from Gen 1

- **Special Stat Split**: The single Special stat is now split into SpA (Special Attack) and SpD (Special Defense)
- **Type Chart Expansion**: Added Dark and Steel types (17 types total), fixing Ghost immunities to Psychic
- **Held Items**: Pokémon can now hold items that provide passive effects during battle
- **Weather Systems**: Rain, Sun, and Sandstorm with turn-based duration and type-specific effects
- **New Status Mechanics**: Sleep duration (1-7 turns), Freeze thaw chance (20% per turn), burn damage, badly poisoned resets on switch
- **Breeding System**: First generation enabling Pokémon reproduction with inheritance mechanics
- **Happiness/Friendship**: Stats and move behavior influenced by friendship value
- **Shiny Pokémon**: Rare color variants determined by DV values (not introduced mid-battle, determined at generation)
- **Time of Day**: In-game time affects move power and evolution
- **Pokérus**: Beneficial virus affecting EXP growth
- **Critical Hit Overhaul**: Changed from Speed-based formula to stage-based (Stage 0 = 1/16 chance, +1 stage = 1/8, etc.)
- **New Moves & Move Categories**: Thief, Mean Look, Protect, Belly Drum, etc.
- **Stat Experience (EVs)**: Foundation for what becomes modern EV system

## 2. Stat System

### DVs (Determinant Values) vs IVs

Gen 2 uses **DVs (0-15 per stat)**, not IVs (0-31). DVs are determined at generation and are **immutable** during play.

### Stat Formula (Gen 1/2 unified)

For **HP**:
```typescript
HP = ((2 * base + DV + EV / 4) * level / 100) + level + 5
```

For **Other Stats**:
```typescript
Stat = ((2 * base + DV + EV / 4) * level / 100) + 5
```

Where:
- `base` = Pokémon's base stat
- `DV` = 0-15 (gen-specific at creation)
- `EV` = Stat Experience points (0-65535 per stat, soft cap ~252)
- `level` = Current level (1-100)

### Shiny Determination (Gen 2)

A Pokémon is **Shiny** if all of the following are true:
- `ATK_DV == DEF_DV`
- `SPD_DV == SPA_DV` (remembering SpA = Sp.Atk in Gen 2)
- `ATK_DV == SPD_DV` (all DVs are equal)
- Additionally, ATK_DV must be `10` or `11` (approximately 1/8192 chance, matching Gen 2 Shiny rate)

Actually, Gen 2's Shiny check is:
```typescript
isShiny(pokémon: Pokémon): boolean {
  const { ATK_DV, DEF_DV, SPD_DV, SPA_DV, SPD_DV } = pokémon.dvs;
  // All attack and defense DVs must be equal, and speed/SpA DVs must be equal
  // Additionally, a XOR of these values against Trainer ID must pass a threshold
  // For simplicity: Shiny occurs when DVs meet specific pattern
  return (ATK_DV === DEF_DV) &&
         (SPD_DV === SPA_DV) &&
         (ATK_DV === SPD_DV) &&
         this.checkTrainerIDShiny(pokémon);
}
```

For implementation, use Bulbapedia's exact algorithm based on Trainer ID XOR.

### Stat Caps

- Base stats cap at 255
- Final stat calculation produces caps naturally via formula
- HP stat formula differs from other stats (see above)

## 4. Damage Formula

### Gen 2 Damage Formula

The Gen 2 damage formula is nearly identical to Gen 1, with the key difference being the Special split:

```typescript
function calculateDamage(
  attacker: Pokémon,
  defender: Pokémon,
  move: Move,
  weather: Weather | null,
  isCritical: boolean,
): number {
  const level = attacker.level;

  // Determine if move is Physical or Special based on TYPE (not move category)
  // In Gen 2, type determines category:
  const isPhysicalType = isPhysicalMoveType(move.type);
  const isSpecialType = isSpecialMoveType(move.type);

  const atkStat = isPhysicalType ? attacker.stats.ATK : attacker.stats.SPA;
  const defStat = isPhysicalType ? defender.stats.DEF : defender.stats.SPD;

  // Base damage calculation
  let damage = ((((2 * level / 5 + 2) * move.power * atkStat) / defStat) / 50) + 2;

  // Critical hit multiplier (always 2x in Gen 2)
  if (isCritical) {
    damage = damage * 2;
  }

  // Type effectiveness (0.5x, 1x, 2x)
  const typeMultiplier = getTypeEffectiveness(move.type, defender);
  damage = damage * typeMultiplier;

  // Weather modifier
  const weatherModifier = getWeatherModifier(move.type, weather);
  damage = damage * weatherModifier;

  // STAB (Same Type Attack Bonus) - 1.5x if attacker shares move type
  const stab = attacker.types.includes(move.type) ? 1.5 : 1.0;
  damage = damage * stab;

  // Random factor: 85-100% (gen 2 specific)
  const randomFactor = 85 + Math.floor(Math.random() * 16); // 85-100 inclusive
  damage = damage * randomFactor / 100;

  // Apply burn modifier if attacker is burned and move is Physical
  if (attacker.status === 'burn' && isPhysicalType) {
    damage = Math.floor(damage * 0.5);
  }

  return Math.max(1, Math.floor(damage));
}

function isPhysicalMoveType(type: PokemonType): boolean {
  const PHYSICAL_TYPES = ['normal', 'fighting', 'flying', 'ground', 'rock', 'bug', 'ghost', 'steel'];
  return PHYSICAL_TYPES.includes(type);
}

function isSpecialMoveType(type: PokemonType): boolean {
  const SPECIAL_TYPES = ['fire', 'water', 'electric', 'grass', 'ice', 'psychic', 'dragon', 'dark', 'poison'];
  return SPECIAL_TYPES.includes(type);
}

function getWeatherModifier(moveType: PokemonType, weather: Weather | null): number {
  if (!weather) return 1.0;

  if (weather.type === 'rain') {
    if (moveType === 'water') return 1.5;
    if (moveType === 'fire') return 0.5;
  } else if (weather.type === 'sun') {
    if (moveType === 'fire') return 1.5;
    if (moveType === 'water') return 0.5;
  } else if (weather.type === 'sandstorm') {
    // Sandstorm does not modify move damage in Gen 2, only deals damage per turn
  }

  return 1.0;
}
```

### Damage Formula Notes

- **Order of operations matters**: Base damage → STAB → Type effectiveness → Weather → Random
- **Caps**: Damage is capped at max HP value; minimum 1 HP
- **Rounding**: Each multiplication is floor-rounded
- **Random factor**: 85-100% range ensures variability in KO thresholds

## 5. Critical Hits

### Critical Stage System

Gen 2 replaced Gen 1's Speed-based crit formula with a **stage-based system**:

```typescript
interface CriticalStage {
  stage: number; // 0, 1, 2, 3
  chance: number; // 1/16, 1/8, 1/4, 1/2
}

const CRIT_CHANCES: Record<number, number> = {
  0: 1 / 16,   // Normal stage
  1: 1 / 8,    // +1 stage
  2: 1 / 4,    // +2 stage
  3: 1 / 2,    // +3 stage
};

function isCriticalHit(attacker: Pokémon, move: Move): boolean {
  const baseStage = 0;
  const moveStageBonus = move.critStage || 0; // Some moves have higher crit
  const totalStage = baseStage + moveStageBonus;

  const chance = CRIT_CHANCES[Math.min(totalStage, 3)];
  return Math.random() < chance;
}
```

### Critical Hit Moves

Certain moves have increased critical hit ratio:
- **Stage +1 moves**: Slash (1/8 base), Bite, Karate Chop, Crabhammer, Leer (lowers defense, separate effect)
- **Stage +2 moves**: Focus Energy (now correctly grants Stage +2, fixing Gen 1 bug where it lowered crit)

### Focus Energy Fix

In Gen 1, Focus Energy **lowered** critical hit chance by mistake. Gen 2 corrects this:

```typescript
class FocusEnergy extends Move {
  execute(user: Pokémon, target: Pokémon): void {
    // Gen 2 correct behavior: raise crit stage by 2
    user.battleStats.critStage = Math.min(user.battleStats.critStage + 2, 3);
  }
}
```

### Critical Multiplier

Critical hits deal **exactly 2x damage** in Gen 2 (no variation).

## 7. Weather Systems

### Weather Types

```typescript
enum WeatherType {
  NONE = 'none',
  RAIN = 'rain',         // Rain Dance
  SUN = 'sun',           // Sunny Day
  SANDSTORM = 'sandstorm', // Sandstorm
  // SNOW/HAIL not in Gen 2 (added Gen 3+)
}

interface Weather {
  type: WeatherType;
  turnsRemaining: number;
  setByMove?: MoveId; // Track which move set it
}
```

### Weather Effects

#### Rain
- **Duration**: 5 turns (set by Rain Dance)
- **Move power modifier**: Water-type moves gain +50% power, Fire-type moves deal -50% damage
- **No automatic damage**: Rain doesn't deal residual damage in Gen 2
- **Visual feedback**: Animation and message each turn

#### Sun / Clear Sky
- **Duration**: 5 turns (set by Sunny Day)
- **Move power modifier**: Fire-type moves gain +50% power, Water-type moves deal -50% damage
- **No automatic damage**: Sun doesn't heal status in Gen 2 (that's Gen 5+)

#### Sandstorm
- **Duration**: 5 turns (set by Sandstorm move, new in Gen 2)
- **Automatic damage**: Non-Rock/Ground/Steel Pokémon take 1/8 max HP damage at end of turn
- **Type of damage**: Fixed recoil, applies before status damage (see: End-of-Turn Order)
- **No move modifier**: Sandstorm doesn't modify move power in Gen 2

### Weather Implementation

```typescript
class WeatherSystem {
  currentWeather: Weather | null = null;

  setWeather(type: WeatherType, duration: number = 5): void {
    this.currentWeather = {
      type,
      turnsRemaining: duration,
    };
  }

  decrementWeather(): void {
    if (this.currentWeather) {
      this.currentWeather.turnsRemaining--;
      if (this.currentWeather.turnsRemaining <= 0) {
        this.currentWeather = null;
      }
    }
  }

  applySandstormDamage(pokémon: Pokémon): void {
    if (this.currentWeather?.type !== WeatherType.SANDSTORM) return;

    const isRockGroundOrSteel = pokémon.types.some(t =>
      ['rock', 'ground', 'steel'].includes(t)
    );

    if (!isRockGroundOrSteel) {
      const damage = Math.ceil(pokémon.stats.HP / 8);
      pokémon.takeDamage(damage, 'sandstorm');
    }
  }
}
```

## 8. Status Conditions

### Sleep

- **Duration**: 1-7 turns (chosen randomly when status applied)
- **Thaw mechanic**: Pokémon wakes up when duration expires OR by Full Heal/Awakening item
- **Immunity**: Wake-Up Slap is a new move that hits harder vs. Sleep

### Paralysis

- **Speed reduction**: Reduces Speed stat to 25% of normal
- **Effect chance**: Some moves (Thunder Wave) paralyze with 100% accuracy in Gen 2 (nerfed in later gens)
- **Cured by**: Paralysis Heal, Full Heal, Cheri Berry (held item), Full Restore

### Poison

#### Regular Poison
- **Damage**: 1/8 max HP per turn
- **Cured by**: Antidote, Full Heal, Pecha Berry (held item), Full Restore

#### Badly Poisoned (Purple Poison)
- **Damage**: Starts at 1/16 max HP, increases by 1/16 each turn (1/16, 2/16, 3/16, etc.) up to max damage
- **Resets on switch**: Counter resets to 1/16 when switching (NEW in Gen 2; Gen 1 persisted across switches)
- **Mechanics**: Badly poisoned status persists, but damage multiplier increases each turn

```typescript
interface BadlyPoisonedStatus {
  counter: number; // 1, 2, 3, ... up to ~16
  damagePerTurn(): number {
    return Math.ceil((this.counter / 16) * maxHP);
  }
  incrementCounter(): void {
    this.counter = Math.min(this.counter + 1, 15); // Cap at reasonable level
  }
  resetOnSwitch(): void {
    this.counter = 1; // NEW in Gen 2
  }
}
```

### Burn

- **Damage**: 1/8 max HP per turn
- **Attack reduction**: Halves Physical Attack stat (ATK)
- **Cured by**: Full Heal, Full Restore, Lum Berry (wait, that's Gen 3+)
- **Note**: Burn doesn't reduce Special Attack in Gen 2 (only ATK)

### Freeze

- **Thaw chance**: 20% chance to thaw each turn at end of turn
- **Immobility**: Frozen Pokémon cannot attack (0% chance to move per turn)
- **Manual thaw**: Can use Full Heal, Full Restore, or other items
- **Weather**: Fire-type moves (attacking) can thaw opponent

### Confusion

- **Duration**: 2-5 turns (random)
- **Chance per turn**: 50% chance to hit self with 40 Base Power Physical attack
- **Damage type**: Counted as Normal-type, not type of move, for STAB/weakness
- **Cured by**: Full Heal, Full Restore, Lum Berry (Gen 3+)

## 9. Move Mechanics

### Move Categories by Type

In Gen 2, **moves are categorized as Physical or Special by their TYPE**, not by the individual move:

```typescript
const PHYSICAL_TYPES = ['normal', 'fighting', 'flying', 'ground', 'rock', 'bug', 'ghost', 'steel'];
const SPECIAL_TYPES = ['fire', 'water', 'electric', 'grass', 'ice', 'psychic', 'dragon', 'dark', 'poison'];

function getMoveCategory(move: Move): 'physical' | 'special' {
  if (PHYSICAL_TYPES.includes(move.type)) return 'physical';
  if (SPECIAL_TYPES.includes(move.type)) return 'special';
  throw new Error(`Unknown type for categorization: ${move.type}`);
}
```

### New Moves (Gen 2 Introduction)

#### Protect
- **Effect**: User is protected from attacks this turn; prevents damage and contact-based effects
- **Multi-turn usage**: Chance of failure after first use (50% on second use, etc.)
- **Priority**: +4 priority (highest in game)

#### Thief
- **Effect**: Deals damage and steals opponent's held item (first move to interact with held items)
- **Interaction**: Stolen item is placed into attacker's bag if inventory space; otherwise stays in battle

#### Attract
- **Effect**: Opposite-gender target becomes attracted; 50% chance to fail action each turn
- **Requirement**: Requires opposite gender; same-gender Pokémon immune
- **Interaction**: Physical/Special moves have no distinction here; move just affects gender

#### Mean Look / Spider Web
- **Effect**: Prevents opponent from switching out; traps on switch
- **Duration**: Persists until move user switches out or battle ends

#### Belly Drum
- **Effect**: User maximizes Attack stat (+6 stage) but takes 50% of max HP as recoil damage
- **Risk/Reward**: High risk but guarantees max Attack

#### Pursuit
- **Effect**: Deals damage; if opponent switches, damage is doubled and move goes first

#### Ancientpower
- **Effect**: 10% chance to raise all stats by 1 stage

### Move Pool Changes

- Many moves re-typed (e.g., Bite becomes Dark-type instead of Normal)
- New Dark and Steel type moves added
- Move power/accuracy adjustments from Gen 1

## 10. New Mechanics

### Happiness / Friendship Evolution

Pokémon have a Happiness value (0-255) that affects:

```typescript
interface Happiness {
  value: number; // 0-255
  maxValue: number = 255;

  // EXP rate modifier
  getExpMultiplier(): number {
    if (this.value >= 200) return 1.05; // +5% EXP
    if (this.value >= 100) return 1.0;  // Normal
    if (this.value < 50) return 0.5;    // -50% EXP (rare)
    return 1.0;
  }

  // Evolution
  canEvolveViaHappiness(pokémonId: string): boolean {
    // Pokémon like Espeon, Umbreon, Pichu evolve at high happiness
    return this.value >= 220; // Typical threshold
```

- **Umbreon**: Eevee evolves to Umbreon at high happiness (220+) when leveled at nighttime
- **Pichu → Pikachu**: Pichu evolves at any happiness when leveled
- **Clefairy (from Cleffa)**: Cleffa evolves at high happiness

### Breeding System

```typescript
interface BreedingResult {
  egg: Pokémon;
  baseSpecies: PokémonId;
  parentMale: Pokémon;
  parentFemale: Pokémon;
}

class BreedingSystem {
  canBreed(pokémon1: Pokémon, pokémon2: Pokémon): boolean {
    // Both must be opposite genders
    // Both must be in same egg group
    // Neither can be legendary/unbreedable
    return pokémon1.gender !== pokémon2.gender &&
           pokémon1.eggGroup === pokémon2.eggGroup &&
           !pokémon1.isLegendary &&
           !pokémon2.isLegendary;
  }

  breed(male: Pokémon, female: Pokémon): BreedingResult {
    // Egg inherits base stats from random parent, IV quality, potential moves
    // Shiny chance based on trainer ID
    const egg = new Pokémon(female.species); // Inherits mother's species
    egg.level = 5; // Starting level
    egg.happiness = 0; // New eggs start with 0 happiness

    // Inherit DVs from parents (50% random from each)
    Object.keys(egg.dvs).forEach(statKey => {
      const randomParent = Math.random() < 0.5 ? male : female;
      egg.dvs[statKey] = randomParent.dvs[statKey];
    });

    return { egg, baseSpecies: female.species, parentMale: male, parentFemale: female };
  }
}
```

```typescript
class ShinySystem {
  generateShiny(trainer: Trainer, wildPokémon: Pokémon): boolean {
    // Shiny chance: 1/8192 in wild encounters
    // Determined by Trainer ID XOR hidden Pokémon ID

    const trainerId = trainer.id & 0xFFFF; // 16-bit trainer ID
    const secretId = trainer.secretId & 0xFFFF; // Gen 2: secret ID
    const xor = trainerId ^ secretId;

    // Check if shiny threshold met
    return xor < (65536 / 8192); // Approximately 1/8192 chance
  }

    // Shiny determination based on DVs in Gen 2
    // (See Stat System section)
    return pokémon.shiny === true;
  }
}
```

## 11. End-of-Turn Order

The exact order of end-of-turn effects is crucial for competitive accuracy:

```typescript
function executeEndOfTurnEffects(battle: Battle): void {
  // Step 1: Process weather damage (Sandstorm)
  for (const pokémon of battle.allActivePokémon()) {
    applyWeatherDamage(pokémon);
  }

  // Step 2: Process status damage (Poison, Burn, Badly Poisoned counter increment)
  for (const pokémon of battle.allActivePokémon()) {
    if (pokémon.status === 'poison') {
      pokémon.takeDamage(Math.ceil(pokémon.stats.HP / 8), 'poison');
    }
    if (pokémon.status === 'burn') {
      pokémon.takeDamage(Math.ceil(pokémon.stats.HP / 8), 'burn');
    }
    if (pokémon.status === 'badly-poisoned') {
      const damage = Math.ceil((pokémon.badlyPoisoned.counter / 16) * pokémon.stats.HP);
      pokémon.takeDamage(damage, 'badly-poisoned');
      pokémon.badlyPoisoned.incrementCounter();
    }
  }

  // Step 3: Process Leech Seed damage (if Gen 2 includes; added Gen 3+)
  // Not present in Gen 2

  // Step 4: Held item effects (Leftovers)
  for (const pokémon of battle.allActivePokémon()) {
    if (pokémon.heldItem?.name === 'Leftovers') {
      const recovery = Math.floor(pokémon.stats.HP / 8);
      pokémon.currentHP = Math.min(pokémon.currentHP + recovery, pokémon.stats.HP);
    }
  }

  // Step 5: Decrement weather turns
  battle.weather.decrementWeather();

  // Step 6: Check for battle end (KO'd Pokémon)
  for (const pokémon of battle.allActivePokémon()) {
    if (pokémon.currentHP <= 0) {
      handleKO(pokémon);
    }
  }
}
```

**Key considerations:**
- Weather damage applies **before** status damage
- Held item effects apply **after** status/weather damage (allows survival checks)
- Status damage triggers before Leech Seed (though LS not in Gen 2)
- Weather duration decrements **after** effects are applied (so weather lasts at least one full turn)

### Gen2Ruleset Architecture

The `Gen2Ruleset` should NOT extend `BaseRuleset` but instead be a standalone implementation implementing `GenerationRuleset`:

### Key Structural Differences

**Gen 1 → Gen 2 Transition**:
- Gen 1: Single `Special` stat
- Gen 2: `Special` splits into `SpA` and `SpD`
- Gen 3+: Move categories change from Type-based to Move-based

**Shared with Gen 1**:
- Stat formula (modified for Special split)
- DV system (not IVs)
- No EVs as modern system (pre-EVs)
- Basic type matchups (17 types added Dark/Steel)

**Changes in Gen 3+**:
- Move categories become individual (Physical/Special per move, not type)
- IVs replace DVs
- EVs become standardized (252/252/4 format)
- Abilities introduced
- New items (Choice Scarf, Life Orb, etc.)
- New weather/terrain systems
