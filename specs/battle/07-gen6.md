# Pokémon Generation 6 (X/Y/ORAS) Battle Mechanics Specification

**Generation**: 6
**Games**: Pokémon X, Y, Omega Ruby, Alpha Sapphire
**Release**: November 2013 (XY), November 2014 (ORAS)
**Base Ruleset**: Extends `BaseRuleset` (Gen 3)

---

## 1. Overview

Generation 6 introduced the most significant metagame shifts since Gen 4 with the addition of the Fairy type, Mega Evolution mechanics, and substantial nerfs to weather permanence. These changes fundamentally reshaped competitive Pokémon.

**Key Features:**
- Fairy type (18th type)
- Mega Evolution battle gimmick (one per battle per team)
- Fairy-type effectiveness overhaul and Steel-type defense loss
- Critical hit multiplier reduction (2x → 1.5x) and rate restructuring
- Weather duration limitation (permanent → 5-8 turns)
- EXP Share rework (held item → key item affecting entire party)
- Pokémon-Amie affection bonus system
- Sky Battles format (Flying-type restriction)
- New entry hazard (Sticky Web)
- Powder/spore immunity for Grass types
- Electric Terrain sleep immunity for grounded Pokémon

## 2. Fairy Type

### Overview

The Fairy type was introduced as the 18th Pokémon type to balance the dominance of Dragon-types and create new strategic opportunities. It represents magical or cute creatures.

### Effectiveness Chart

**Fairy is super-effective against:**
- Dragon
- Fighting
- Dark

**Fairy resists:**
- Fire
- Poison
- Steel

**Fairy is immune to:**
- Dragon

**Super-effective against Fairy:**
- Fire
- Poison
- Steel

### Offensive Buffs

The addition of Fairy type provided offensive coverage benefits to two existing types:

**Poison type:**
- Now resists Fairy in addition to previous resistances
- Poison-type moves deal super-effective damage to Fairy (new)

**Steel type:**
- Now resists Fairy in addition to previous resistances
- Steel-type moves deal super-effective damage to Fairy (new)

### Notable Type Changes

Several Pokémon had their types changed or reassigned to Fairy in Gen 6:
- Clefairy line → Fairy (was Normal)
- Jigglypuff line → Normal/Fairy (was Normal)
- Mr. Mime and Mr. Rime → Psychic/Fairy
- Togetic line → Fairy/Flying
- Azurill line → Water/Fairy (was Water/Normal)
- Audino → Normal/Fairy
- Marill line → Water/Fairy (was Water/Normal)
- Various legendary Pokémon received Fairy typing

## 4. Mega Evolution

### Concept

Mega Evolution is a temporary battle transformation that enhances a Pokémon's stats and may change its type or ability. It represents the pinnacle of human-Pokémon bonding and is central to Gen 6's competitive meta.

**Rules:**
- Each team can Mega Evolve one Pokémon per battle (not per turn)
- Mega Evolution occurs at the start of the turn, before moves are executed
- Requires a held Mega Stone specific to that Pokémon and evolution
- Lasts until the Pokémon is switched out or faints
- Can be undone if the Pokémon switches out (but can't be used again in battle)
- Priority: Mega Evolution happens in turn order based on speed

### Implementation

#### BattleGimmick Interface

```typescript
interface BattleGimmick {
  readonly name: string;
  readonly description: string;
  canActivate(pokemon: Pokemon, battle: Battle): boolean;
  activate(pokemon: Pokemon, battle: Battle): void;
  deactivate(pokemon: Pokemon, battle: Battle): void;
  isActive(pokemon: Pokemon): boolean;
}
```

#### MegaEvolution Class

```typescript
class MegaEvolution implements BattleGimmick {
  readonly name = 'Mega Evolution';
  readonly description = 'Transform a Pokémon into its Mega form, enhancing stats and possibly changing type/ability.';

  private activePokemon: Map<string, MegaSpecies> = new Map();
  private usedInBattle: Map<string, boolean> = new Map();

  canActivate(pokemon: Pokemon, battle: Battle): boolean {
    // Check if Pokémon has already mega evolved in this battle
    if (this.usedInBattle.get(battle.id)) {
      return false;
    }

    // Check if Pokémon's team has already used mega evolution
    const teamMegaUsed = Array.from(this.usedInBattle.entries())
      .filter(([pokemonId, used]) =>
        used && battle.isPokemonInTeam(pokemonId, pokemon.teamId)
      );

    if (teamMegaUsed.length > 0) {
      return false;
    }

    // Check if holding appropriate Mega Stone
    const megaStone = pokemon.heldItem;
    if (!megaStone || !megaStone.isMegaStone) {
      return false;
    }

    // Verify Mega Stone matches Pokémon
    const megaSpecies = getMegaEvolution(pokemon.species, megaStone);
    return megaSpecies !== null;
  }

  activate(pokemon: Pokemon, battle: Battle): void {
    const megaStone = pokemon.heldItem;
    if (!megaStone?.isMegaStone) return;

    const megaSpecies = getMegaEvolution(pokemon.species, megaStone);
    if (!megaSpecies) return;

    // Store current state for potential reversion
    const originalSpecies = pokemon.species;
    const originalAbility = pokemon.ability;
    const originalStats = { ...pokemon.baseStats };

    // Apply Mega Evolution
    pokemon.species = megaSpecies;
    pokemon.baseStats = megaSpecies.stats;

    if (megaSpecies.ability) {
      pokemon.ability = megaSpecies.ability;
    }

    if (megaSpecies.type) {
      pokemon.types = megaSpecies.type;
    }

    // Recalculate stats
    pokemon.recalculateStats();

    // Mark as mega evolved and used
    this.activePokemon.set(pokemon.id, megaSpecies);
    this.usedInBattle.set(pokemon.id, true);

    battle.broadcast('mega-evolution', {
      pokemonId: pokemon.id,
      pokemonName: pokemon.nickname || pokemon.species.name,
      megaForm: megaSpecies.name,
    });
  }

  deactivate(pokemon: Pokemon, battle: Battle): void {
    // Revert to base form when switching out
    // Note: Cannot be re-activated once deactivated
    if (this.activePokemon.has(pokemon.id)) {
      const originalData = pokemon.megaData;
      pokemon.species = originalData.baseSpecies;
      pokemon.baseStats = originalData.baseStats;
      pokemon.ability = originalData.baseAbility;
      pokemon.types = originalData.baseTypes;
      pokemon.recalculateStats();

      this.activePokemon.delete(pokemon.id);
    }
  }

  isActive(pokemon: Pokemon): boolean {
    return this.activePokemon.has(pokemon.id);
  }

  resetForBattle(battle: Battle): void {
    this.activePokemon.clear();
    this.usedInBattle.clear();
  }
}
```

## 5. Critical Hit Changes

### Multiplier Reduction

**Gen 3-5:** Critical hits dealt 2x damage
**Gen 6+:** Critical hits deal 1.5x damage

This significant reduction had major competitive implications, making critical-reliant strategies less reliable.

### Critical Hit Stage Rates

The probability of scoring a critical hit is now determined by a 5-stage system with significant changes from previous generations.

| Stage | Gen 5 Rate | Gen 6 Rate | Description |
|-------|-----------|-----------|------------|
| 0 | 1/16 | 1/24 | Base rate, reduced |
| 1 | 1/8 | 1/8 | +1 Crit Stage |
| 2 | 1/4 | 1/2 | +2 Crit Stages |
| 3+ | 1/3 | Guaranteed | +3 or more Crit Stages |

### Abilities Affected

**Abilities that increase crit rate become less dominant:**
- Sniper (boost was 1.5x damage on crits, now 1.5 × 1.5 = 2.25x)
- Super Luck (+1 crit stage, less powerful now)

### Moves Affected

**High-crit moves lose value but remain viable:**
- Night Slash, Psychic, Scope Lens equipped
- Stone Edge, Slash, Cross Chop
- Focus Energy (+2 crit stage, now guaranteed at 2+ stages)

### Implementation

```typescript
class CriticalHit {
  // Gen 6 multiplier
  static readonly DAMAGE_MULTIPLIER = 1.5; // Was 2.0 in Gen 5

  static calculateCritRate(pokemon: Pokemon, move: Move): number {
    let baseRate = 1/24; // Stage 0

    if (pokemon.critStage >= 3) {
      return 1.0; // Guaranteed crit
    }

    const critRates = [
      1/24,  // Stage 0
      1/8,   // Stage 1
      1/2,   // Stage 2
      1.0,   // Stage 3+
    ];

    const stage = Math.min(pokemon.critStage, 3);
    return critRates[stage];
  }

  static isCriticalHit(pokemon: Pokemon, move: Move): boolean {
    const critRate = this.calculateCritRate(pokemon, move);
    return Math.random() < critRate;
  }

  static applyDamageModifier(baseDamage: number, isCrit: boolean): number {
    if (!isCrit) return baseDamage;
    return Math.floor(baseDamage * this.DAMAGE_MULTIPLIER);
  }
}
```

## 6. Weather Nerfs

### Duration Changes

One of the most impactful changes in Gen 6, weather abilities were limited to stop their permanent prevalence.

| Condition | Gen 5 Duration | Gen 6 Duration (Natural) | Gen 6 Duration (Rock) |
|-----------|----------------|--------------------------|----------------------|
| Rain (Drizzle) | Permanent | 5 turns | 8 turns |
| Sun (Drought) | Permanent | 5 turns | 8 turns |
| Hail | Permanent | 5 turns | 8 turns |
| Sandstorm | Permanent | 5 turns | 8 turns |

**Weather Rock Items:**
- Damp Rock (extends Rain)
- Heat Rock (extends Sun)
- Smooth Rock (extends Hail)
- Assault Vest (WAIT, WRONG - Assault Vest is not a weather rock)
- Icy Rock (extends Hail)
- Smooth Rock (extends Hail)
- Soft Sand (NOT a weather rock)

The weather rocks extend the duration by 3 turns (5 → 8 turns total).

### Strategic Impact

- Eliminates permanent weather-based teams that dominated Gen 5
- Reduces value of weather-dependent Pokémon
- Creates more turn-limited strategic windows
- Forces teams to plan around weather duration
- Abilities like Drought/Drizzle become turn-limited setups

### Affected Abilities

**Abilities that summon weather (now limited):**
- Drizzle (Kyogre)
- Drought (Groudon)
- Hail (Abomasnow)
- Sand Stream (Tyranitar, Hippowdon)

**Abilities that benefit from weather:**
- Swift Swim, Chlorophyll, Sand Rush, Dry Skin, etc. (now more situational)

### Implementation

```typescript
class WeatherSystem {
  private activeWeather: Weather | null = null;
  private weatherTurnsRemaining: number = 0;
  private weatherSource: 'ability' | 'move' | 'terrain' = 'move';

  // Gen 6 constants
  static readonly WEATHER_DURATION_NATURAL = 5;
  static readonly WEATHER_DURATION_WITH_ROCK = 8;

  setWeather(
    type: Weather,
    source: 'ability' | 'move' | 'terrain',
    pokemon?: Pokemon
  ): void {
    this.activeWeather = type;
    this.weatherSource = source;

    // Determine duration based on held item
    let duration = WeatherSystem.WEATHER_DURATION_NATURAL;
    if (pokemon?.heldItem?.isWeatherRock) {
      duration = WeatherSystem.WEATHER_DURATION_WITH_ROCK;
    }

    this.weatherTurnsRemaining = duration;

    // Log weather activation
    console.log(`${type} activated for ${duration} turns`);
  }

  decrementWeatherTimer(): void {
    if (this.weatherTurnsRemaining > 0) {
      this.weatherTurnsRemaining--;

      if (this.weatherTurnsRemaining === 0) {
        console.log(`Weather (${this.activeWeather}) faded.`);
        this.activeWeather = null;
      }
    }
  }

  getActiveWeather(): Weather | null {
    return this.activeWeather;
  }

  isWeatherActive(type: Weather): boolean {
    return this.activeWeather === type && this.weatherTurnsRemaining > 0;
  }
}
```

## 7. EXP Share Change

### Mechanic Rework

**Gen 3-5:** EXP Share was a held item that gave EXP to the holder only
**Gen 6+:** EXP Share is a Key Item that automatically distributes EXP to all party members

### Distribution

- All Pokémon in the party receive experience from battles
- Default state: EXP Share is ON (can be toggled off in settings)
- Distribution method: All Pokémon receive equal EXP (not split among them)
- Accounts for levels differently: lower-level Pokémon may receive multiplier bonuses

### Implementation

```typescript
interface EXPShareKeyItem extends KeyItem {
  id: 'exp-share';
  isEnabled: boolean;
  distributeExperience(
    expGained: number,
    participant: Pokemon,
    partyMembers: Pokemon[]
  ): Map<Pokemon, number>;
}

class EXPShare implements EXPShareKeyItem {
  id = 'exp-share' as const;
  name = 'EXP Share';
  description = 'When activated, all Pokémon in your party receive EXP from battles.';
  isEnabled: boolean = true;

  distributeExperience(
    expGained: number,
    participant: Pokemon,
    partyMembers: Pokemon[]
  ): Map<Pokemon, number> {
    const distribution = new Map<Pokemon, number>();

    if (!this.isEnabled) {
      // Old behavior - only participant gets EXP
      distribution.set(participant, expGained);
      return distribution;
    }

    // Gen 6+ behavior - all get equal EXP
    for (const member of partyMembers) {
      // Account for level differences if desired
      const expForMember = this.calculateExpForMember(
        expGained,
        member.level,
        participant.level
      );
      distribution.set(member, expForMember);
    }

    return distribution;
  }

  private calculateExpForMember(
    baseExp: number,
    memberLevel: number,
    participantLevel: number
  ): number {
    // Simple implementation: all members get equal exp
    // Can be enhanced with level-based modifiers
    return baseExp;
  }

  toggleEnabled(): void {
    this.isEnabled = !this.isEnabled;
  }
}
```

## 8. Affection (Pokémon-Amie)

### Overview

Pokémon-Amie is a mechanic that increases affection with Pokémon through petting, feeding, and playing minigames. In battle, affection grants several performance bonuses.

### Affection Bonuses

**1. Experience Bonus**
- Affectionate Pokémon gain 20% extra EXP from battles
- Applies to all EXP sources

**2. Critical Hit Rate Increase**
- Affectionate Pokémon gain +1 to critical hit stage for all moves
- Combined with move-specific crit stages

**3. Survival Bonus (Important)**
- Affectionate Pokémon survive moves that would be lethal (1 HP recovery)
- Occurs once per battle (or limited uses)
- Activation message: "[Pokémon]'s love for you pulled it through!"

**4. Status Condition Recovery**
- Affectionate Pokémon may spontaneously cure status conditions
- Approximately 10-15% chance per turn when afflicted
- Affects: poison, paralysis, burn, sleep, freeze

### Affection Levels

| Level | Name | Bonuses |
|-------|------|---------|
| 0 | No Affection | None |
| 1-2 | Neutral | N/A |
| 3-4 | Friendly | EXP +10% |
| 5+ | Very Friendly | EXP +20%, Crit +1, Survival, Status recovery |

### Implementation

```typescript
class Affection {
  static readonly MAX_AFFECTION = 255;
  static readonly LEVEL_THRESHOLD = [0, 70, 140, 210];

  private affectionValue: number = 0;
  private survivalUsesRemaining: number = 1;

  constructor(initialAffection: number = 0) {
    this.affectionValue = Math.min(initialAffection, Affection.MAX_AFFECTION);
  }

  getAffectionLevel(): number {
    if (this.affectionValue < 70) return 0;
    if (this.affectionValue < 140) return 1;
    if (this.affectionValue < 210) return 2;
    return 3;
  }

  // Battle bonuses
  getExpBonus(): number {
    const level = this.getAffectionLevel();
    if (level >= 3) return 1.2; // +20%
    if (level >= 2) return 1.1; // +10%
    return 1.0;
  }

  getCritStageBonus(): number {
    return this.getAffectionLevel() >= 3 ? 1 : 0;
  }

  canSurviveLethalHit(): boolean {
    return this.getAffectionLevel() >= 3 && this.survivalUsesRemaining > 0;
  }

  applySurvivalBonus(pokemon: Pokemon): void {
    if (this.canSurviveLethalHit()) {
      pokemon.currentHP = 1;
      this.survivalUsesRemaining--;
      console.log(
        `${pokemon.nickname || pokemon.species.name}'s love for you pulled it through!`
      );
    }
  }

  shouldCureStatus(): boolean {
    return (
      this.getAffectionLevel() >= 3 &&
      Math.random() < 0.125 // 12.5% chance
    );
  }

  increaseAffection(amount: number = 1): void {
    this.affectionValue = Math.min(
      this.affectionValue + amount,
      Affection.MAX_AFFECTION
    );
  }

  decreaseAffection(amount: number = 1): void {
    this.affectionValue = Math.max(this.affectionValue - amount, 0);
  }

  resetSurvivalUsesForBattle(): void {
    this.survivalUsesRemaining = 1;
  }
}
```

### Pokémon-Amie Activities

```typescript
class PokemonAmie {
  private affectionMap: Map<Pokemon, Affection> = new Map();

  petPokemon(pokemon: Pokemon): void {
    if (!this.affectionMap.has(pokemon)) {
      this.affectionMap.set(pokemon, new Affection());
    }
    const affection = this.affectionMap.get(pokemon)!;
    affection.increaseAffection(10);
  }

  feedPokemon(pokemon: Pokemon, poffin: Poffin): void {
    if (!this.affectionMap.has(pokemon)) {
      this.affectionMap.set(pokemon, new Affection());
    }
    const affection = this.affectionMap.get(pokemon)!;
    affection.increaseAffection(poffin.affectionBoost);
  }

  playMinigame(pokemon: Pokemon, minigame: Minigame): void {
    if (!this.affectionMap.has(pokemon)) {
      this.affectionMap.set(pokemon, new Affection());
    }
    const affection = this.affectionMap.get(pokemon)!;

    if (minigame.playerWon) {
      affection.increaseAffection(20);
    } else {
      affection.increaseAffection(5);
    }
  }

  getAffection(pokemon: Pokemon): Affection {
    if (!this.affectionMap.has(pokemon)) {
      this.affectionMap.set(pokemon, new Affection());
    }
    return this.affectionMap.get(pokemon)!;
  }
}
```

## 9. Sky Battles

### Overview

Sky Battles are a format restriction introduced in Gen 6 where only Flying-type Pokémon or Pokémon with a Flying-type move can participate. These occur in specific locations (e.g., cliff edges, bridges).

### Rules

- Only Pokémon with Flying type can participate
- Non-Flying Pokémon cannot be used (will refuse to battle)
- Special encounter format with unique visual presentation
- Primarily single battles

### Implementation

```typescript
interface BattleFormat {
  id: string;
  name: string;
  canParticipate(pokemon: Pokemon): boolean;
}

class SkyBattle implements BattleFormat {
  id = 'sky-battle';
  name = 'Sky Battle';

  canParticipate(pokemon: Pokemon): boolean {
    return pokemon.types.includes('flying');
  }

  validateTeam(team: Pokemon[]): { valid: boolean; reason?: string } {
    if (!team.every(p => this.canParticipate(p))) {
      return {
        valid: false,
        reason: 'Sky Battles can only be entered with Flying-type Pokémon.',
      };
    }
    return { valid: true };
  }
}
```

## 10. Sticky Web

### Overview

Sticky Web is a new entry hazard introduced in Gen 6 that reduces the Speed of Pokémon entering the field.

### Mechanics

- Entry hazard (like Spikes, Stealth Rock)
- Affects single layer only (no accumulation)
- Reduces Speed of opposing Pokémon by 1 stage when switching in
- Affects all Pokémon that switch in, regardless of type

### Effect Details

| Property | Value |
|----------|-------|
| Layer Limit | 1 |
| Speed Reduction | -1 stage |
| Affects | All Pokémon except those with immunity |
| Removal | Rapid Spin, Defog |
| Priority | Applied on switch-in |

### Moves that Set Sticky Web

- Sticky Web (Grass-type status move)
- Signature move of certain Pokémon

### Implementation

```typescript
class StickyWeb extends EntryHazard {
  readonly id = 'sticky-web';
  readonly name = 'Sticky Web';
  readonly type = 'grass';
  readonly layers = 1; // Single layer only

  onSwitchIn(pokemon: Pokemon, field: Field): void {
    // Check immunity abilities
    if (pokemon.ability.hasImmunity('entry-hazard:sticky-web')) {
      return;
    }

    // Reduce Speed by 1 stage
    pokemon.stats.speedStage = Math.max(pokemon.stats.speedStage - 1, -6);

    console.log(
      `${pokemon.nickname || pokemon.species.name}'s Speed was lowered by Sticky Web!`
    );
  }

  canBeRemoved(): boolean {
    return true; // Removed by Rapid Spin, Defog
  }
}

class Field {
  private entryHazards: Map<string, EntryHazard> = new Map();

  setStickyWeb(team: 'player' | 'opponent'): void {
    const hazard = new StickyWeb();
    this.entryHazards.set(`${team}:sticky-web`, hazard);
  }

  hasStickyWeb(team: 'player' | 'opponent'): boolean {
    return this.entryHazards.has(`${team}:sticky-web`);
  }

  removeStickyWeb(team: 'player' | 'opponent'): void {
    this.entryHazards.delete(`${team}:sticky-web`);
  }
}
```

## 12. Powder/Spore Immunity

### Overview

In Gen 6, Grass-type Pokémon became immune to powder and spore-based moves.

### Affected Moves

**Powder/Spore moves that now don't affect Grass types:**
- Powder Snow
- Stun Spore
- Sleep Powder
- Poison Powder
- Spore
- Leech Seed (note: Leech Seed already didn't affect Grass in previous gens)
- Cotton Guard (WAIT, Cotton Guard is not a powder move)
- Worry Seed
- Effect Spore (ability)

**Notable Grass types protected:**
- Venusaur (immune to Sleep Powder, Stun Spore, etc.)
- Celebi
- Shaymin
- Whimsicott

### Implementation

```typescript
class Move {
  isPowderMove(): boolean {
    return ['powder-snow', 'stun-spore', 'sleep-powder', 'poison-powder', 'spore', 'worry-seed'].includes(
      this.id
    );
  }

  canAffect(target: Pokemon): boolean {
    // Grass types are immune to powder/spore moves
    if (
      this.isPowderMove() &&
      target.types.includes('grass')
    ) {
      return false;
    }

    // Check other immunities
    return this.checkOtherImmunities(target);
  }
}

class Pokemon {
  canBeAffectedByPowderMove(move: Move): boolean {
    if (!move.isPowderMove()) return true;
    return !this.types.includes('grass');
  }
}
```

## 13. Electric Terrain Immunity

### Overview

When Electric Terrain is active, Pokémon resting on the ground (grounded Pokémon) cannot fall asleep.

### Mechanics

- Electric Terrain (new terrain in Gen 6+)
- Prevents sleep for grounded Pokémon
- Does not prevent other status conditions (paralysis, etc.)
- Boosts Electric-type moves by 1.3x

### Affected Abilities

- Static (may still paralyze when struck)
- Dry Skin (still effective for healing)

### Implementation

```typescript
class Terrain {
  readonly id: string;
  readonly name: string;

  applyTerrainEffects(pokemon: Pokemon, field: Field): void {
    // Subclasses override
  }
}

class ElectricTerrain extends Terrain {
  readonly id = 'electric-terrain';
  readonly name = 'Electric Terrain';

  applyTerrainEffects(pokemon: Pokemon, field: Field): void {
    // Boost Electric-type moves
    if (pokemon.types.includes('electric')) {
      pokemon.stats.spAtkMultiplier = 1.3;
    }
  }

  canFall asleep(pokemon: Pokemon): boolean {
    // Grounded Pokémon cannot fall asleep on Electric Terrain
    if (this.isGrounded(pokemon)) {
      return false;
    }
    return true;
  }

  private isGrounded(pokemon: Pokemon): boolean {
    return (
      pokemon.types.includes('electric') ||
      pokemon.ability.id === 'static' ||
      pokemon.heldItem?.providesGrounding
    );
  }
}
```

**Note:** The actual grounding calculation depends on whether the Pokémon is floating, flying, or has an ability like Magnet Rise. Electric Terrain specifically affects Pokémon that are physically grounded.

## 14. Implementation Notes

### Gen6Ruleset Class

```typescript
class Gen6Ruleset extends BaseRuleset {
  readonly generation = 6;
  readonly name = 'Generation 6 (X/Y/ORAS)';

  // Initialize Gen 6 specific systems
  private megaEvolution: MegaEvolution;
  private typeChart: TypeChart;
  private weatherSystem: WeatherSystem;
  private terrainSystem: TerrainSystem;
  private affectionSystem: PokemonAmie;

  constructor() {
    super();
    this.megaEvolution = new MegaEvolution();
    this.typeChart = this.buildGen6TypeChart();
    this.weatherSystem = new WeatherSystem();
    this.terrainSystem = new TerrainSystem();
    this.affectionSystem = new PokemonAmie();
  }

  private buildGen6TypeChart(): TypeChart {
    const chart = super.getTypeChart(); // Get Gen 3 base

    // Add Fairy type
    chart.addType('fairy', {
      superEffectiveAgainst: ['dragon', 'fighting', 'dark'],
      resistedBy: ['poison', 'steel'],
      superEffectiveAgainstBy: ['fire', 'poison', 'steel'],
      resistsAttackFrom: ['fighting', 'bug', 'dark'],
      immuneTo: ['dragon'],
      immuneAgainst: [],
    });

    // Steel loses Ghost and Dark resistances
    const steelData = chart.getType('steel');
    steelData.resistedBy = steelData.resistedBy.filter(
      type => !['ghost', 'dark'].includes(type)
    );

    return chart;
  }

  // Ruleset overrides for Gen 6 mechanics
  initializeBattle(battle: Battle): void {
    super.initializeBattle(battle);
    this.megaEvolution.resetForBattle(battle);
  }

  resolveTurn(battle: Battle, turn: Turn): void {
    // Mega Evolution phase (before moves)
    this.resolveMegaEvolutions(battle, turn);

    // Standard turn resolution
    super.resolveTurn(battle, turn);

    // Weather timer decrement (new in Gen 6)
    this.weatherSystem.decrementWeatherTimer();
  }

  private resolveMegaEvolutions(battle: Battle, turn: Turn): void {
    const activePokemon = [
      battle.getActivePokemon(0),
      battle.getActivePokemon(1),
    ];

    const sortedBySpeed = activePokemon.sort(
      (a, b) => b.stats.speed - a.stats.speed
    );

    for (const pokemon of sortedBySpeed) {
      if (this.megaEvolution.canActivate(pokemon, battle)) {
        this.megaEvolution.activate(pokemon, battle);
      }
    }
  }

  calculateDamage(
    attacker: Pokemon,
    defender: Pokemon,
    move: Move,
    field: Field
  ): number {
    let damage = super.calculateDamage(attacker, defender, move, field);

    // Critical hit multiplier reduced to 1.5x
    if (this.isCriticalHit(attacker, move)) {
      damage = Math.floor(damage * 1.5); // Was 2.0 in Gen 5
    }

    return damage;
  }

  applyEXP(
    pokemonList: Pokemon[],
    baseEXP: number,
    expShare: EXPShare
  ): void {
    const distribution = expShare.distributeExperience(
      baseEXP,
      pokemonList[0],
      pokemonList
    );

    for (const [pokemon, exp] of distribution) {
      const affectionBonus = this.affectionSystem
        .getAffection(pokemon)
        .getExpBonus();
      const finalExp = Math.floor(exp * affectionBonus);

      pokemon.addExperience(finalExp);
    }
  }

  // Override methods that changed in Gen 6
  getWeatherDuration(source: 'ability' | 'move'): number {
    return WeatherSystem.WEATHER_DURATION_NATURAL; // 5 turns
  }

  getCriticalHitMultiplier(): number {
    return 1.5; // Down from 2.0
  }

  getCriticalHitRate(pokemon: Pokemon, move: Move): number {
    const baseRate = move.critRate || 1/24;
    const affectionBonus =
      this.affectionSystem.getAffection(pokemon).getCritStageBonus();

    return this.calculateCritRate(pokemon, move, affectionBonus);
  }
}
```

```typescript
// Example: Converting a Gen 5 team to Gen 6
function migrateTeamFromGen5(gen5Team: Pokemon[]): Pokemon[] {
  return gen5Team.map(pokemon => {
    const gen6Pokemon = pokemon.clone();

    // Check for type changes
    gen6Pokemon.types = this.getGen6Types(pokemon.species);

    // Check for ability changes on Mega Evolutions
    if (this.canMegaEvolve(gen6Pokemon)) {
      gen6Pokemon.ability = this.getGen6Ability(gen6Pokemon);
    }

    // Reset moves that changed significantly (Earthquake, etc.)
    gen6Pokemon.moves = this.validateGen6Moveset(gen6Pokemon.moves);

    // Initialize affection if needed
    if (!gen6Pokemon.affection) {
      gen6Pokemon.affection = new Affection(0);
    }

    return gen6Pokemon;
  });
}
```
